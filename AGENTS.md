# AGENTS 总入口

本文件是 AI 在本项目中的唯一总入口。每次对话自动加载，不需要手动 @。

---

## 一、项目与创作者

**张半蛋**，自媒体创作者，双平台（抖音+小红书）口播短视频。

**一句话定位**：8 年教育人，教普通人用 AI 学得更快、学得更明白、学完真能用。

**背景**：8 年互联网教育公司运营经验（3 岁到初中用户全覆盖），AI 重度使用者，一个人全流程，每天晚上 8-10:30 创作。

---

## 二、读取顺序

1. 先读本文件 `AGENTS.md`，判断任务类别与全局约束。
2. 如果是项目相关讨论、决策、规划，先读 `对话备忘录.md`。
3. 再按任务类型调用对应的 Codex Skill。Codex 可从 `.agents/skills/` 自动发现这些 Skills。
4. 如果一个任务跨多个类型，可以同时调用多份 Skills，但以本文件中的全局规则为先。
5. `.cursor/rules/` 仅作为 Cursor 兼容副本保留，不再作为 Codex 的规则入口。

## 四、全局协作规则

- 先给结论，再展开细节。
- 保持口语化，像同事协作，不写过度书面化表达。
- 讨论内容选题、脚本和账号方向时，优先判断用户真正想表达什么、长期关注什么、哪些内容适合张半蛋讲；不要优先迎合观众、追热点或套爆款模板。
- 数据是参考，不是方向盘。完播率、涨粉、点赞、收藏、评论用于辅助判断观众是否听懂、表达是否清楚，不直接决定后续内容方向。
- 如果需求不清楚，先问再做，不要靠猜。
- 保持简洁，能一句话说清就不要写很长。
- 如果判断不确定，要明确说明不确定点和原因。
- 如果用户当前要求与项目内既有记录、已确认状态或既有规则冲突，必须先确认，不能直接覆盖。

## 五、重要约束

1. **短视频封面文案 / 脚本输出前必须先过 dbs 核查**——先用 `$dbs-content` 判断选题是否真实想表达、是否适合张半蛋讲、形式、认知落差、表达效率、封面标题方向；开头不稳再用 `$dbs-hook`，标题封面再用 `$dbs-xhs-title`。先输出 dbs 判断和建议方向，等用户确认后，才能输出封面文案或完整脚本。
2. **脚本输出前必须先去 AI 味**——调用 `$humanizer-zh`，不能跳过。
3. **与现有记录/已确认状态冲突时，必须先问用户确认**，不能直接覆盖。
4. **学到新结论/策略/决策时，主动提醒是否写入** `对话备忘录.md`。
5. **数据复盘降权**——每周或每两周做一次整体复盘即可，复盘重点是表达是否被理解、开头是否留住、评论是否有真实共鸣；不要因为账号前期小样本数据频繁调整方向。
6. **家庭教育内容暂缓**——等教育标签立住、家长群体有自然聚集信号后再启动。
7. **表达习惯**：不用「过几天就散了/散掉了」，用「当下会觉得懂了，过几天基本就忘了」。
8. **抖音审核风控**：发布前检查标题、封面、标签、前 10 秒台词，避免「裁员 / 被裁 / AI 替代 / 公司在砍」等就业焦虑词同时高密度出现；默认不用 `#裁员`，优先用「职场变化 / 岗位变化 / 这次不一样」降敏。

## 七、常用工作流 SOP

### 新内容：从灵感到发布

```
1. 有灵感 → 记入 知识库/灵感速记.md（30秒）
2. 评估选题 → 调用 $cp-topic-eval
3. 确认值得做 → 在 知识库/选题池.md 建条目，确定栏目线
4. 写脚本 → 在 作品/YYYYMMDD_[选题名]/ 下新建脚本.md
             先调用 $dbs-content 核查选题、形式、封面标题方向
             → 调用 $dbs-xhs-title 核查封面标题公式和风险表达
             → 用户确认 dbs 判断后，调用 $cp-script-draft 出初稿
             → 脚本文档默认只保留：发布信息、封面/标题/标签、脚本文案（拍摄版/剪映分批录制）
             → 去AI味处理（调用 $humanizer-zh）
5. 发布前 → 做抖音审核风险自检（就业焦虑词、裁员词、敏感标签、前10秒风险词密度）
6. 录制/剪辑 → 参照脚本_VX.md 的「脚本文案（拍摄版，剪映分批录制）」
7. 发布后 → 提取本条素材（调用 $cp-material-extract）
           → 在 知识库/选题池.md 标记为 ✅ 已发布
8. 周日晚 → 先在两个后台各导出一次新 xlsx 放入 `运营数据/[平台]/`
              → 调用 $data-weekly 生成本周或双周复盘报告
              → 数据只作为表达反馈参考，不直接决定下周内容方向
```

### 快速找选题

```
1. 先扫 知识库/灵感速记.md
2. 再看 知识库/选题池.md 的储备选题
3. 先判断最近真正想表达、真正关注的问题是什么
4. 调用 $cp-topic-eval 评估「是否适合我讲」和表达角度
5. 不确定选哪个 → 优先选表达欲更强、真实案例更足、长期更想沉淀的内容；数据表现只作为辅助参考
```

### 旧内容翻新

```
1. 找 6 个月前的高互动脚本（收藏/分享率高的）
2. 确认是否适配当前 3 条栏目线
3. 按当前口语习惯重写，植入教育经验，输出新版本
4. 新建文件夹：作品/YYYYMMDD_[原选题名]_翻新版/
```

## 八、Skills 使用规则

### 当前可用 Skills


| Skill                  | 触发词               | 功能                      |
| ---------------------- | ----------------- | ----------------------- |
| `$cp-script-draft`     | 帮我写脚本/出初稿         | 基于选题生成脚本初稿，只保留发布信息、标题封面和拍摄版 |
| `$cp-topic-eval`       | 帮我评估这个选题/这个题目值得做吗 | 评估选题潜力，给推荐度和理由          |
| `$cp-material-extract` | 帮我提取素材/发完了提取一下    | 从已完成脚本提取钩子/金句/Prompt/封面 |
| `$data-weekly`         | 周报复盘/本周复盘/跑一下本周数据 | 读取两平台最新 xlsx，产出本周或双周数据参考与表达观察 |
| `$humanizer-zh`                | 去AI味/这段改一下        | 口播脚本去 AI 化处理            |
| `$dbs-xhs-title`       | 帮我起标题             | 生成小红书标题                 |
| `$dbs-hook`            | 帮我优化开头/开头怎么写      | 诊断+优化短视频开头              |
| `$dbs-content`         | 帮我看看这条内容          | 整体内容诊断                  |
| `$dbs-diagnosis`       | 帮我诊断这个问题          | 商业/内容/执行问题诊断            |
| `$zbd-content-workflow` | 内容创作相关任务          | 应用项目级内容策略、脚本和双平台规则       |
| `$zbd-deep-reading`     | 深度阅读/精读材料         | 扫描、拆解并讨论书籍、文章、报告和 PDF    |
| `$zbd-learning-quiz`    | 学习讲解/学完后测验        | 控制讲解、测验、笔记和学习进度流程        |
| `$zbd-session-memory`   | 项目讨论/策略决策         | 读取并按确认更新跨对话备忘录            |
| `$zbd-work-assistant`   | 数据分析/复盘/周报         | 处理运营数据、项目复盘和工作型笔记         |


### 目录与管理

- Codex 统一从 `.agents/skills/` 发现所有项目 Skills。
- 内容生产 Skills（`cp-*`）和项目工作流 Skills（`zbd-*`）直接放在 `.agents/skills/`。
- 诊断工具箱源码仍放在 `.agents/dbs/`，通过 `.agents/skills/dbs*` 软链接供 Codex 发现。
- 去 AI 味工具源码仍放在 `.agents/去AI味工具/`，通过 `.agents/skills/humanizer-zh` 软链接供 Codex 发现。
- 新增或删除 Skill 后，更新 `.agents/skills/README.md`；Skill 的 `name` 与文件夹名保持一致。

## 九、项目记忆与备忘录

- 每次新对话开始，只要涉及项目讨论、决策、规划，先读取 `对话备忘录.md`。
- 当讨论出了新的策略、方向、决策、待办或流程调整时，主动提醒用户是否写入 `对话备忘录.md`。
- 写入备忘录时只记结论，不记过程；一条结论一行，并保留日期。
- 已经落实到具体文件或规则中的内容，不再重复写进备忘录，必要时标记为「已归档」。
- 纯操作性请求、已经直接写入其他文件的内容、学习过程中的知识点，不写入备忘录。
