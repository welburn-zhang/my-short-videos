# 短视频项目

个人短视频内容创作工作区，覆盖选题策划、脚本撰写、发布复盘、素材沉淀、学习研究和 AI 协作规则。

## 核心入口

| 文件 | 给谁看 | 负责什么 |
| --- | --- | --- |
| `AGENTS.md` | AI 自动读取 | 项目总规则、任务路由、SOP、Skills 索引、文件导航 |
| `对话备忘录.md` | AI + 人 | 当前仍有效的策略、方向、流程共识 |
| `README.md` | 人 | 项目目录说明、命名规范、日常查找入口 |
| `.agents/skills/` | AI | 内容生产、学习、复盘、深度阅读等项目 Skills |
| `.agents/dbs/` | AI | dontbesilent 诊断工具箱源码 |
| `.cursor/rules/` | 兼容保留 | Cursor 旧规则副本，不再作为 Codex 主入口 |

## 目录分组

```text

├── README.md              # 人看的项目说明
├── 对话备忘录.md            # 跨对话关键结论
│
├── 作品/                   # 每条内容的脚本、封面、素材、成片
├── 知识库/                 # 定位、选题池、灵感、工作流、创作要求
├── 素材库/                 # 可复用钩子、金句、案例、Prompt、封面标题
├── 脚本模板库/              # 稳定脚本框架
├── 复盘/                   # 人看的阶段复盘、表达观察、策略判断
│
├── 运营数据/               # 平台导出的原始数据和生成报告
├── 学习笔记/               # 学习笔记、测验报告、学习进度
├── 行业报告/               # 外部报告、资料和摘要
```

## 目录职责

### `作品/`

一条内容一个目录，命名为 `YYYYMMDD_选题简称/`。日期使用创建日期或开始写作日期，不强求发布日期。

常见结构：

```text
YYYYMMDD_选题简称/
├── 脚本.md               # 初始稿或主稿
├── 脚本_V2.md            # 迭代稿
├── 选题说明.md            # 选题判断、表达角度、素材依据
├── 数据观察.md            # 单条内容发布后的表达反馈
├── 封面1.jpg
├── 定稿.mov              # 发布成片，默认不进 Git
└── 视觉动效_V3/           # 单条内容的演示、动效或辅助工程
```

补充约定：

- 新内容先建在 `作品/YYYYMMDD_选题简称/`。
- 新建作品后，同步更新 `作品/README.md`。
- 脚本默认使用 `脚本.md`；多版本用 `脚本_V1.md`、`脚本_V2.md`。
- 发布状态以用户确认和平台后台为准，文件里有 `定稿.mov` 只能说明本地存在成片。

### `知识库/`

放项目主线相关的长期文档，适合经常回看：

- 账号定位与方向规划
- IP 定位分析
- 灵感速记
- 选题池
- 内容创作要求
- 每周工作流
- 剪辑模板库

和账号创作主线关系不强的学习计划，优先放到 `学习笔记/` 或另建更明确的位置。

### `复盘/` 与 `运营数据/`

二者分工不同：

- `复盘/`：放人看的判断，重点是表达是否被理解、开头是否留住、评论是否有真实共鸣。
- `运营数据/`：放平台导出的 xlsx、html 报告和数据处理产物。

不要因为单条数据频繁改方向；数据只作为表达反馈。

## 命名规范

| 类型 | 命名 |
| --- | --- |
| 作品目录 | `YYYYMMDD_选题简称/` |
| 脚本版本 | `脚本.md`、`脚本_V1.md`、`脚本_V2.md` |
| 选题说明 | `选题说明.md` |
| 单条数据观察 | `数据观察.md` |
| 运营数据 | `开始日期_至今_平台_日数据_v版本号.xlsx` |
| 周期复盘 | `YYYYMMDD_复盘主题.md` |

## 常用工作流

1. 有灵感：记入 `知识库/灵感速记.md`。
2. 评估选题：调用 `$cp-topic-eval`，判断是否值得做、适不适合张半蛋讲。
3. 确认值得做：在 `知识库/选题池.md` 建条目。
4. 写脚本：在 `作品/YYYYMMDD_选题名/` 下创作，先过 dbs 前置核查。
5. 录制剪辑：参照脚本里的拍摄版或剪映分批录制版。
6. 发布前：检查标题、封面、标签和前 10 秒风险表达。
7. 发布后：调用 `$cp-material-extract` 提取可复用素材。
8. 周期复盘：导出双平台数据到 `运营数据/`，再生成 `复盘/` 文档。

## 整理原则

- 先更新索引和说明，再移动真实文件。
- 不把视频、图片、字体、压缩包、安装包、数据导出文件放进 Git。
- 已归档但仍有参考价值的内容，保留在原目录或索引中标注，不急着删除。
- 新规则优先写入 `AGENTS.md` 或对应 Skill；README 只做导航，不承载复杂规则。
