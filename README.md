# 短视频项目

个人短视频内容创作工作区，覆盖选题策划、脚本撰写、发布复盘、素材沉淀与工作辅助。

## 核心入口


| 文件                    | 给谁看      | 负责什么                             |
| --------------------- | -------- | -------------------------------- |
| `AGENTS.md`           | AI（自动加载） | 任务路由、全局约束、文件导航、SOP 工作流、Skills 索引 |
| `.cursor/rules/*.mdc` | AI（自动加载） | 各任务类型的具体规则                       |
| `对话备忘录.md`            | AI + 人   | 阶段性策略、定位、变现等跨对话关键结论              |
| `README.md`（本文件）      | 人        | 项目说明、目录结构、命名规范、文件约定              |

## 命名规范

- 选题目录：`YYYYMMDD_选题简称/`
- 日期使用开始日期或创建日期，不强求发布日期
- 选题简称尽量简短，保证一眼能识别
- 运营数据文件：`开始日期_至今_平台_日数据_v版本号.xlsx`

---

## 作品目录约定

- **浏览顺序**：按时间线倒序（最新在前）。终端可 `ls -1 作品 | sort -r` 查看。
- **索引导航**：完整列表见 `作品/README.md`（新选题记得插在列表顶部）。

一个常见的选题目录包含：

```text
YYYYMMDD_选题简称/
├── 脚本.md           # 初始版本
├── 脚本_V2.md        # 迭代版本
├── 封面1.jpg
├── 定稿.mov          # 发布成品
└── 素材.mov          # 拍摄原片
```

补充约定：

- 脚本默认使用 `脚本.md`，迭代用 `脚本_VX.md`
- 封面图按版本编号，如 `封面1.jpg`、`封面2.jpg`
- `.gitignore` 默认忽略 `*.mov`，核对发布状态时优先看文件路径或让创作者确认

## 常用工作流

1. **有灵感** → 记入 `知识库/灵感速记.md`（30 秒能完成）
2. **评估选题** → 在 Cursor 里调用 `/cp-topic-eval`
3. **确认值得做** → 在 `知识库/选题池.md` 建条目
4. **写脚本** → 在 `作品/YYYYMMDD_[选题名]/` 下创作
5. **录制/剪辑** → 参照脚本中的剪辑标注
6. **发布后** → 调用 `/cp-material-extract` 提取素材
7. **每周日** → 复制 `复盘/模板_每周复盘.md` 做数据复盘
