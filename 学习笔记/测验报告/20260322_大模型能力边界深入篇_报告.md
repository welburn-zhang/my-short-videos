## 学习主题

大模型能力边界与局限（深入篇）— 幻觉分类、上下文虚标、Jagged Frontier、Prompt 脆弱性、推理模型本质、Token 成本经济学、技术方案选型。

## 知识要点回顾

1. **幻觉三分类**：事实性（编造事实）、忠实性（与给定资料矛盾）、推理性（逻辑跳跃）
2. **Lost in the Middle**：大模型对长文本中间内容遗忘严重，有效上下文远小于标称值
3. **Jagged Frontier**：AI 能力边界是锯齿形，不能凭直觉判断某功能是否可靠
4. **Prompt 脆弱性**：措辞、顺序、示例数量、模型版本都会影响输出，系统 prompt 需要版本管理
5. **推理模型**：通过思维链提升准确率，但成本和延迟翻倍，不适合所有场景
6. **Token 经济学**：AI 产品边际成本不为零，模型选择是商业决策
7. **方案选型**：知识常更新用 RAG，知识稳定用 Fine-tuning，控制风格用 Prompt Engineering

## 测验题目与作答

### 第 1 题
**用户给 AI 一份正确的合同文档，AI 在总结时却把金额搞错了。这属于哪种类型的幻觉？**

| 选项 | 内容 |
|------|------|
| A | 事实性幻觉 — 编造不存在的事实 |
| **B ✅** | **忠实性幻觉 — 回答与提供的资料矛盾** |
| C | 推理性幻觉 — 逻辑链条中某一步跳跃 |
| D | 这不算幻觉，属于模型 bug |

> ✅ 正确。资料是正确的，但模型的回答与资料矛盾，这是忠实性幻觉的典型表现。

---

### 第 2 题
**关于大模型的长上下文能力，以下哪项描述是正确的？**

| 选项 | 内容 |
|------|------|
| A | 标称支持 128K token 就意味着 128K 范围内的信息都能被准确利用 |
| **B ✅** | **大模型对开头和结尾的内容记得好，中间内容容易被忽略（Lost in the Middle）** |
| C | 上下文窗口越大，回答质量一定越高 |
| D | RAG 检索时应该尽量多塞文档，充分利用上下文窗口 |

> ✅ 正确。Lost in the Middle 是已被研究验证的现象，有效上下文远小于标称上下文。

---

### 第 3 题
**「参差不齐的前沿」（Jagged Frontier）这个概念说的是什么？**

| 选项 | 内容 |
|------|------|
| A | 不同公司的 AI 模型能力差距很大 |
| **B ✅** | **AI 的能力边界是锯齿形的 — 有些难的事做得好，有些简单的事反而做不好** |
| C | AI 的发展速度时快时慢，不是线性进步 |
| D | AI 在不同语言上的表现参差不齐 |

> ✅ 正确。这意味着不能凭直觉判断"这个功能 AI 能不能做好"，必须做充分边界测试。

---

### 第 4 题
**关于推理模型（如 o1、DeepSeek-R1），以下哪项是错误的？**

| 选项 | 内容 |
|------|------|
| A | 推理模型通过 Chain-of-Thought 先"思考"再回答，提升准确率 |
| B | 推理模型的 token 消耗和延迟都会大幅增加 |
| **C ✅** | **推理模型适合所有场景，应该全面替代普通模型** |
| D | 推理模型对简单问题可能过度思考，反而答错 |

> ✅ 正确。推理模型成本高、延迟大，简单场景反而可能过度思考答错，需要模型路由策略。

---

### 第 5 题
**一个 AI 客服产品日活 10 万，每人每天对话 3 次。关于成本，以下哪个认知是正确的？**

| 选项 | 内容 |
|------|------|
| A | 和传统互联网产品一样，多一个用户边际成本几乎为零 |
| **B ✅** | **每多一个用户、每多一次对话都会产生推理成本，边际成本不为零** |
| C | 只要用免费的开源模型，就不需要考虑推理成本 |
| D | 输入 token 和输出 token 的价格是一样的 |

> ✅ 正确。这是 AI 产品和传统互联网产品最本质的商业模型区别，直接影响定价策略。

---

### 第 6 题
**企业知识库的内容经常更新，想让 AI 基于最新知识回答用户问题，应该优先选择哪个方案？**

| 选项 | 内容 |
|------|------|
| A | Fine-tuning — 把知识训练进模型 |
| B | Prompt Engineering — 把知识写在提示词里 |
| **C ✅** | **RAG — 实时检索知识库再生成回答** |
| D | 换一个更大的模型就能解决 |

> ✅ 正确。知识经常更新 → RAG（知识库随时可更新）。Fine-tuning 适合知识稳定不常变的场景。
