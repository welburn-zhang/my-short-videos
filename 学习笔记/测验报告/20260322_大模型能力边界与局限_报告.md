## 学习主题

大模型能力边界与局限 — 系统了解大模型本质原理、RAG、Agent、多模态等核心概念及各自的能力边界。

## 知识要点回顾

1. **大模型本质**：文字接龙（预测下一个词），不是真正理解，会产生幻觉
2. **RAG**：先检索再生成，解决大模型不了解私有知识/实时信息的问题
3. **Agent**：大模型 + 工具 + 规划，能执行任务但目前不够可靠
4. **多模态**：文字之外处理图像/语音/视频的能力，但精度和成本仍有局限
5. **能力边界**：文本生成可靠，精确计算/事实准确性/物理世界操控不可靠

## 测验题目与作答

### 第 1 题
**大模型生成内容的本质原理是什么？**

| 选项 | 内容 |
|------|------|
| A | 从数据库中查找最匹配的答案返回给用户 |
| **B ✅** | **根据前面的文字，预测下一个最可能出现的词（文字接龙）** |
| C | 像人一样理解语义后，组织语言回答 |
| D | 从互联网实时搜索信息后整理回答 |

> ✅ 正确。大模型的核心就是"next token prediction"，理解这个本质才能理解它为什么会幻觉、为什么需要 RAG 等补充方案。

---

### 第 2 题
**RAG（检索增强生成）主要解决大模型的什么问题？**

| 选项 | 内容 |
|------|------|
| A | 让大模型运行速度更快 |
| B | 让大模型能处理图片和视频 |
| **C ✅** | **让大模型能基于特定知识库回答问题，减少幻觉** |
| D | 让大模型能自主调用外部工具完成任务 |

> ✅ 正确。RAG 的"检索"环节先找到相关文档，再喂给大模型生成答案，核心解决的就是幻觉和私有知识问题。D 选项描述的是 Agent 的能力。

---

### 第 3 题
**以下关于 Agent（智能体）的描述，哪一项是错误的？**

| 选项 | 内容 |
|------|------|
| A | Agent 能调用外部工具（如搜索引擎、API） |
| B | Agent 能把复杂任务拆解成多个步骤执行 |
| **C ✅** | **目前 Agent 完成复杂多步骤任务已经非常可靠，成功率接近 100%** |
| D | Agent 的多步推理会消耗更多 Token，成本更高 |

> ✅ 正确。这是错误选项。Agent 目前最大的问题就是不可靠，多步任务中任何一环出错就会崩，71.4% 企业智能体平台用不好就是证据。

---

### 第 4 题
**当企业想搭建一个「基于公司内部文档的智能客服」时，最核心需要用到的技术方案是？**

| 选项 | 内容 |
|------|------|
| A | 多模态 |
| B | Agent |
| **C ✅** | **RAG** |
| D | 微调（Fine-tuning） |

> ✅ 正确。基于企业内部文档的问答就是 RAG 最经典的应用场景：用户提问 → 检索内部文档 → 大模型基于文档生成回答。

---

### 第 5 题
**以下哪项是大模型目前确实做不好/不可靠的？**

| 选项 | 内容 |
|------|------|
| A | 写一篇 500 字的产品介绍文案 |
| B | 将一段英文翻译成中文 |
| **C ✅** | **精确计算 7 位数乘法并保证 100% 正确** |
| D | 对一段文字做摘要总结 |

> ✅ 正确。文案、翻译、摘要都是大模型的强项。但精确数学计算是大模型的已知弱点，因为它本质是在"预测文字"而不是在"做数学运算"。
