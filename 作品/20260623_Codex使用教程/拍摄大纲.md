# Codex 使用教程｜拍摄大纲

> 这是一份拍摄结构，不是逐字稿。录制时可以根据实际界面和操作情况自由发挥。
>
> 功能核对时间：2026 年 6 月 23 日。Codex 更新较快，发布前再检查一次界面名称。

## 一、这条视频要解决什么问题

帮助第一次打开 Codex 的普通用户看懂整个界面，知道每个区域有什么用，以及应该从哪里开始。

这条只讲入门地图，不详细展开插件、Skill、自动化和移动端的具体玩法，后续再分别拍摄。

---

## 二、推荐讲解顺序

### 1. 真人出镜：先讲 Codex 是什么

简单说明 Codex 的特色，以及为什么普通人也可以使用它。

**大白话解释：**普通聊天 AI 主要负责回答问题，Codex 不只会回答，还可以读取资料、修改文件、使用工具，继续把一个任务往下做。

### 2. 下载与登录

#### 2.1 下载 Codex

说明 Codex 支持 macOS 和 Windows。

**大白话解释：**先根据自己的电脑系统下载对应版本，安装方式和普通软件差不多。

#### 2.2 登录账号

可以使用 ChatGPT 账号登录，也可以使用 OpenAI API Key。

**大白话解释：**大部分普通用户直接登录自己的 ChatGPT 账号就可以开始使用。

#### 2.3 网络环境

需要能够正常访问 OpenAI 服务的网络环境。

**大白话解释：**如果 OpenAI 服务本身无法连接，软件里的登录和任务执行也会受到影响。

### 3. 认识首页的三个核心区域

先快速指出三个区域，不急着立刻讲完所有细节：

1. 左侧功能区。
2. 项目与对话区。
3. 中间的对话工作区。

**大白话解释：**左边负责找入口，中间选择在哪里工作，右边完成这一次具体任务。

---

## 三、左侧功能区

### 4. 搜索

可以查找过去创建的对话。

**大白话解释：**以前让 Codex 做过什么，可以直接搜索，不用一条条往下翻。

### 5. 插件与 Skill

这一条只讲基本区别，不展开安装和制作过程。

#### 5.1 插件

插件可以为 Codex 增加外部工具、应用连接和新的能力。

**大白话解释：**插件就像给 Codex 安装新的工具，让它能去更多地方读取信息或者执行操作。

#### 5.2 Skill

Skill 是一套可以被 Codex 重复执行的任务说明和工作流程。

**大白话解释：**Skill 就像提前写好的工作说明书，以后遇到同类任务，不用每次从头教。

### 6. 自动化

自动化可以让 Codex 按照设置好的时间重复执行任务。

**大白话解释：**原来需要你定期提醒 Codex 做的事情，现在可以让它到时间自己开始。

> 提醒：项目自动化通常要求电脑保持开机和联网、Codex 正在运行，并且项目文件夹仍然可用。

### 7. 移动端

通过 ChatGPT 手机 App，可以连接一台正在运行 Codex 的电脑。

**大白话解释：**人在外面时，也可以用手机给家里的电脑下任务、看进度和处理审批。

> 提醒：实际的文件读取和命令执行仍然发生在连接的电脑上，电脑需要保持可用。

---

## 四、项目与对话区

### 8. 项目

项目对应电脑中的一个工作文件夹，Codex 会围绕这个文件夹读取和修改文件。

**大白话解释：**项目决定 Codex 这次在哪里工作，以及它主要能看到哪些资料。

### 9. 新建项目

现场演示新建一个文件夹，再把它添加到 Codex。

**大白话解释：**先给 Codex 准备一个专门的工作空间，相关资料都可以放进这个文件夹。

### 10. 新建对话

同一个项目下面可以建立多个对话，每个对话负责不同任务。

**大白话解释：**项目是同一个办公室，对话是办公室里分别推进的不同工作。

> 不同对话的聊天记录相互独立，但同一项目中的多个对话可能修改同一批文件，不代表文件完全隔离。

### 11. 不绑定项目的 Chat

还可以不选择项目，直接创建普通 Chat。

**大白话解释：**如果只是研究问题、整理想法或者调用外部工具，不需要处理某个文件夹，可以直接聊天。

### 12. 侧边聊天

可以针对当前回复或执行过程，在侧边继续追问。

**大白话解释：**想问进度、让它解释某一步，又不想打断主任务，就在旁边单独问一句。

---

## 五、对话输入区域

### 13. “+”号与附件

可以上传图片、文档和其他任务需要的材料。

**大白话解释：**只靠文字说不清楚时，直接把资料交给 Codex 看。

### 14. 聊天框里的三个常用符号

#### 14.1 `/` 斜杠

用来打开系统命令菜单，例如计划模式、目标模式和状态查询；菜单中也可能显示已启用的 Skill。

**大白话解释：**不知道功能藏在哪里，先输入斜杠，很多常用命令都能在这里找到。

#### 14.2 `@`

用来引用插件、外部工具或者应用，例如 Browser、Chrome 和 Computer Use。

**大白话解释：**想明确告诉 Codex 这次要使用哪个外部工具，就输入 `@` 去选择。

#### 14.3 `$`

用来直接点名调用某个 Skill。

**大白话解释：**已经知道要用哪套工作流程，就输入 `$` 把对应的 Skill 叫出来。

### 15. 计划模式

复杂任务开始前，可以先让 Codex 了解资料、追问问题并整理执行步骤。

**大白话解释：**先别急着动手，让 Codex 把准备怎么做说清楚，你确认后再开始。

### 16. 目标模式

为较长的任务设置一个持续推进的目标和完成标准。

**大白话解释：**告诉 Codex 最后必须做到什么，它会围绕这个结果持续工作，而不是做一步就停下来。

### 17. 审批权限

可以设置 Codex 能访问和修改哪些内容，以及什么操作必须先获得确认。

**大白话解释：**决定哪些事情 Codex 可以自己做，哪些事情必须先问过你。

### 18. 模型与思考深度

可以根据任务选择模型，并调整推理深度或速度。

**大白话解释：**简单任务可以让它快一点，复杂任务可以给它更多时间想清楚。

### 19. 语音输入

可以通过语音输入需求，再把转写后的文字发给 Codex。

**大白话解释：**需求很长、不想打字时，可以直接说出来，检查一下文字再发送。

## 六、右上角工作面板

### 20. 文件

可以查看当前项目的目录和具体文件内容。

**大白话解释：**想知道项目里有哪些资料，或者 Codex 新建了什么文件，就来这里看。

### 21. 审查

可以查看 Git 项目当前有哪些文件变化，以及具体修改了哪些内容。

**大白话解释：**Codex 说改完了先别急着信，在这里看它到底动了哪些文件、改了哪几行。

> 提醒：这里显示的是项目中的 Git 变更，可能包括 Codex 的修改，也可能包括用户自己尚未提交的修改。

### 22. 终端

可以使用电脑本地的命令行，运行程序、测试和 Git 命令。

**大白话解释：**很多原来需要程序员手动敲命令完成的事情，Codex 可以在这里帮你执行和检查。

### 23. 浏览器

可以在 Codex 中打开网页、预览页面、截图和添加注释。

**大白话解释：**看到哪里需要修改，直接圈出来或者写一句备注，AI 会更容易理解你的要求。

特别适合演示：

- 预览刚刚生成的前端页面。
- 截取当前页面。
- 选中元素或圈出区域。
- 给字体、颜色、间距和布局添加修改意见。
- 让 Codex 根据这些注释继续调整 UI。

> 如果网页依赖原有浏览器的登录状态、Cookie 或扩展，使用 Codex 的 Chrome 扩展会更合适。

### 24. 侧边聊天

右侧也可以展开侧边聊天，对当前任务进行补充追问。

**大白话解释：**主任务还在进行时，可以在旁边问一个小问题，不会把主线带跑偏。

## 七、设置

### 25. 工作模式

#### 25.1 适用于编程

会展示更多技术性的回复和控制信息。

**大白话解释：**想看清代码、命令和具体执行细节，就选择编程模式。

#### 25.2 适用于日常工作

保持相同能力，但减少技术细节，用更容易理解的方式呈现结果。

**大白话解释：**不关心代码怎么写，只想整理文档、研究资料或者把事情做完，就选择日常工作模式。

> 两种模式不是切换模型，也不是两套完全不同的 Codex。底层能力没有因此变弱，主要区别是展示多少技术细节。日常工作模式仍然可以处理代码，编程模式也可以处理文档。

### 26. 语言

可以切换 Codex 的界面语言。

**大白话解释：**这只是改变软件菜单和按钮显示什么语言，不会限制你用中文还是英文提问。

### 27. 外观主题

可以选择浅色、深色或跟随系统，也可以继续调整界面颜色。

**大白话解释：**根据自己的使用习惯，把 Codex 调成看着最舒服的样子。

### 28. 字体与字号

可以调整界面字体、代码字体和字号。

**大白话解释：**录屏时可以把字调大一点，观众看操作会更清楚。

### 29. 个性化风格

可以选择 Friendly、Pragmatic 或关闭默认个性风格。

**大白话解释：**你可以让 Codex 说话更友好、更直接，或者只按照自己的指令来。

### 30. 自定义指令

可以设置希望 Codex 长期遵守的个人表达和协作偏好。

**大白话解释：**把你每次都要重复说的习惯提前告诉 Codex，以后新对话也能按照这些要求配合你。

> 自定义指令主要是个人层面的长期偏好；具体项目还可以通过项目里的 `AGENTS.md` 设置单独规则。

### 31. Codex 宠物

可以选择一个浮动宠物，并通过它查看任务状态。

**大白话解释：**它不只是桌面装饰，还会提醒你 Codex 正在工作、等待回复，还是已经完成。

### 32. Computer Use

安装并授权后，Codex 可以看见并操作指定的电脑软件。

**大白话解释：**除了处理文件和命令，Codex 还可以像人一样点击按钮、输入内容和切换页面。

> macOS 首次使用通常需要开启屏幕录制和辅助功能权限；Codex 使用具体 App 时仍可能需要用户确认。

### 33. Git

Git 用来管理本地项目的修改记录、版本和分支。

**大白话解释：**改错了可以看记录，多个版本也不容易混在一起。

### 34. GitHub

连接 GitHub 后，可以同步远端代码仓库、推送代码并处理 Pull Request。

**大白话解释：**Git 管本地版本，GitHub 负责把项目放到线上，并和别人一起协作。

> 不连接 GitHub，Codex 也可以操作电脑中的本地项目；Git 和 GitHub 不是同一个东西。

## 八、结尾总结

帮助新手记住四件事：

1. 先选择 Codex 要工作的项目文件夹。
2. 在对话中说清任务，并提供需要的资料。
3. 根据任务选择计划、目标、权限、模型和工具。
4. 完成后查看文件和审查面板，确认结果是否真的能用。

**大白话解释：**会新建项目、会下任务、会选择工具、会检查结果，就已经能够开始使用 Codex 了。

最后预告后续内容：

- Codex 插件具体怎么使用。
- Skill 怎么安装、创建和调用。
- 自动化任务怎么设置。
- 移动端怎么连接电脑。
- Codex 如何解决真实工作和生活问题。

## 九、拍摄表达提醒

- 每介绍一个功能，先操作，再用一句大白话解释它能解决什么问题。
- 不需要把每个设置都完整配置一遍，重要功能现场演示，次要功能快速带过。
- 设置和菜单可以适当加速，项目、对话、浏览器批注和审查过程需要放慢。
- 可以使用“像带一个新同事”的类比，但不要让类比盖过真实界面。
- 大头画面不要遮挡点击区域，讲到右侧面板时可以临时缩小或移动位置。
- 插件、Skill、自动化和移动端只建立印象，给后续单独视频留出空间。
