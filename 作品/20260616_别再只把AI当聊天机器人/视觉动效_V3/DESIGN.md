# 张半蛋视觉动效规范 V1

## Style Prompt

蓝色轻科技与教育讲解感结合。画面要清楚、有秩序、有一点技术感，但不能像软件发布会、游戏 HUD 或 AI 工具广告。浅色画面负责解释，深蓝画面负责章节切换，橙色只用于错误、风险和关键转折。动效的任务是带着观众看重点，不是炫技。

## Colors

| 角色 | 色值 | 用法 |
| --- | --- | --- |
| 科技蓝 | `#3478F6` | 编号、进度、主强调 |
| 深海蓝 | `#112A4D` | 章节卡、深色背景 |
| 浅雾蓝 | `#EEF5FF` | 浅色背景、信息区 |
| 强调橙 | `#FF9D42` | 错误、警示、转折 |
| 墨蓝黑 | `#182230` | 正文、深色文字 |
| 柔白 | `#F8FBFF` | 深色背景文字 |

## Typography

- 中文标题：`PingFang SC`，800-900 字重，紧凑字距。
- 中文正文：`PingFang SC`，400-600 字重。
- 数字和步骤编号：`JetBrains Mono`，700 字重，使用等宽数字。
- 不使用机械感过强的未来字体，也不使用霓虹描边字。

## Motion

- 章节切换：0.3-0.5 秒，方向明确，优先使用推入、遮罩和块面覆盖。
- 关键词强调：0.2-0.4 秒，使用划线、圈画、局部放大。
- 信息出现顺序必须与口播顺序一致。
- 一个画面只保留一个主要动作，最多再加一个环境层缓慢运动。
- 大字报动效结束后停留至少 1.2 秒，保证观众能读完。

## Visual Grammar

| 内容含义 | 固定表现 |
| --- | --- |
| 新章节 | 深蓝背景 + 大编号 + 白色标题 + 蓝色推进线 |
| 错误结果 | 浅灰蓝卡片 + 橙色圈画或叉线 |
| 正确结果 | 白色卡片 + 科技蓝边框 + 完成状态 |
| 三个步骤 | 蓝色编号卡依次点亮 |
| 前后对比 | 左侧橙色“不能用”，右侧蓝色“能执行” |
| 资料领取 | 文档卡片侧向弹出 + 蓝色关键词标签 |

## Avatar

- 默认放左下角，直径约占画面宽度的 15%-18%。
- 录屏重点与左下区域冲突时移动到左上角。
- 使用白色外圈和科技蓝细描边，不加霓虹光。
- 头像只用于保持人物存在感，不抢操作界面的注意力。

## What NOT to Do

- 不使用黑底霓虹蓝、紫蓝渐变、密集数据流。
- 不堆扫描线、粒子、代码雨和无意义 HUD 元素。
- 不让每个字都跳动，不连续使用弹簧或夸张回弹。
- 不让橙色和蓝色同时大面积争抢注意力。
- 不把人物处理成“技术主播悬浮窗”，人物仍然是可信的讲解者。
