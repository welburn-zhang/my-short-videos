# 脚本 V2：别再只把 AI 当聊天机器人了

## 选题信息

- **主轴**：做事轴
- **类型**：AI 使用方式 / 职场提效 / 任务型 AI
- **展现形式**：真人口播 + 豆包办公任务录屏
- **状态**：V2 待确认
- **核心观点**：普通人用 AI 的下一步，不是多问几个提示词，而是把真实任务说清楚，让 AI 帮你拆任务、跟任务、交结果。
- **录屏主案例**：豆包“办公任务”做短视频选题调研 / 活动复盘 / 会议纪要整理。
- **风险规避**：不做豆包工具测评，不做 Agent 技术教程，不制造“不会 Agent 就落后”的焦虑。

## 脚本文案（剪映分批录制版）

我最近使用AI发现一个问题。以前我们用豆包、DeepSeek这类工具，大部分时间还是在聊天。你问一句，它答一句。你让它写个文案，它就写好发给你，你让它总结资料，就发给你一段话。它更像一个特别聪明的聊天窗口。可是一到真实工作里，很多事情不是问一句就能解决的。比如做一次复盘、整理一份报告、准备一场面试、甚至规划一段学习计划，它里面都有好几步，不是一个答案就能结束的。

我自己之前用 AI，也经常会卡在这。比如我跟它说，帮我做一次复盘。它很快就给我一套模板，看起来是挺完整的，但当我真正去用的时候，还是要自己把资料一份份翻出来，再一点点放进去。所以后来我慢慢意识到，问题不在于 AI 会不会回答，而在于我有没有把这件事说成一个可以执行的任务，然后ai自己就把活干了。

最近豆包新出的这个“办公任务”模式，这个模式和传统的模式，最大的区别是，不是让你继续跟 AI 一问一答，而是更像你把一个任务交给它，然后告诉它最后要什么结果，豆包就自己去干了，像编辑文档，信息收集，写代码这些工作，现在豆包都可以做了。那更专业一点的人，可能会直接去用 Codex、Claude Code 这类 Agent，做更复杂的项目，甚至让它改代码、跑流程。但如果刚开始接触 AI，其实没必要一上来就搞得那么复杂。网络、账号、英文界面、环境配置，任何一个地方卡住，学习的意愿度就会降低。普通人可以先用一个门槛低一点的工具，体验一下“AI 真正帮你干活”是什么感觉，就够了。

我拿一个我做自媒体的场景举例。比如我要做一份短视频的选题调研，我会和AI说：我是一个做 AI 学习和职场内容的创作者，受众是 大学生和在职普通人。请围绕“AI 从聊天机器人变成任务助理”这个方向，帮我做一份短视频选题调研。里面要包括用户痛点、用户在关注什么、适合拍摄的选题、还有哪些选题不建议做。

我其实就是把背景、目标、受众和最后要交付的东西说清楚，然后AI自己就去干活了。

其实使用AI很像我在公司带一个新人，你就说一句“你去分析一下用户反馈”，新人肯定不知道从哪儿开始。但你说清楚，分析哪一批用户，看哪些问题，最后整理成什么表，他就能往下做。AI 也差不多。你给了一句含糊其辞的话，它就只能给你一个很泛的回答。你把任务边界说清楚，它才有机会做的更好。AI帮你干活的过程，我觉得挺像跟一个新人一起改方案。第一版不一定准，你要边看边改。这样用下来，AI 就不是只回你几句话，而是真的能帮你解决一件复杂的事情。

做会议纪要也一样，不要和AI说“帮我总结会议”，而是说，这是一场项目复盘会，请按已完成事项、当前卡点、责任人、下周动作，进行整理。

准备面试也一样，不是说“帮我模拟面试”，而是说我投的是某某岗位，这是这个岗位的介绍，请根据我的简历，帮我列出一面面试常见的问题、可能追问和回答风险点。

我们普通人用 AI，可以先试着和AI聊天慢慢升级到让AI帮你干活，不一定上来就用很复杂的 Agent，也不用一开始就研究一堆技术名词，先找一件自己最近要做的事，把背景讲清楚，把资料给AI，把最后想要的结果说出来，剩下的交给AI，但它给的东西不能直接全信，尤其是数据、调研和专业判断，还是要自己再审核一遍。但它至少能先帮你把一团乱麻、很繁琐的事情梳理出来。对很多工作和学习场景来说，这一步已经很有用了。
