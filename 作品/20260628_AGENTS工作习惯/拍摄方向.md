# C21｜AGENTS.md 是什么

## 拍摄定位

- **系列位置**：Codex 官方最佳实践第 4 条，前面三条讲单次任务，这条开始讲长期项目协作。
- **核心问题**：AI Agent 不只是聊天，它会读文件、改文件、推进任务，所以必须先知道这个项目里的文件规则和工作习惯。
- **一句话方向**：AGENTS.md 不是普通说明文档，而是写给 AI Agent 的「家规 + 厨房地图 + 冰箱使用说明」。
- **核心表达**：AI Agent 不是不会干活，它是太会干活了，所以你更得先告诉它规矩。

## 切入点和方向

- 不再从「重复沟通很烦」这种痛点慢慢铺，也不从概念定义开场。
- 开头参考「秋芝2046」三条样本：轻松打招呼或直接切入，先抛强判断 / 强结果 / 强情绪，再用生活化荒诞比喻制造画面，15 秒内把主题拉回 AGENTS.md。
- 这条建议开头：
  - 你好啊朋友们，今天这期可能会让你的 AI Agent 少犯一半的傻。
  - 因为很多人用 Agent 的方式，就像请人来家里做饭，但不告诉他米在哪、盐在哪、哪口锅能用。
  - 它不是不会干活，它是太会干活了，所以你更得先告诉它规矩。
  - 这个规矩，在 Codex 里就叫 AGENTS.md。
- 重点不是讲 `AGENTS.md` 这个文件名，而是讲清楚：
  - AI Agent 为什么需要项目规则。
  - AGENTS.md 相当于什么。
  - 为什么要用好它。
  - 它大概该怎么用。

## 核心类比

AGENTS.md 可以类比成「你家厨房的使用说明书」。

想象你请了一个特别能干的人来家里帮你做饭。这个人刀工很好，动作很快，还特别主动。但如果你什么都不交代，他就只能靠猜：

- 哪个柜子里是米，哪个盒子里是盐。
- 冰箱里哪些菜是今晚要吃的，哪些菜先不要动。
- 该用哪瓶酱油。
- 哪个锅可以炒菜，哪个锅只能煮汤。
- 吃完以后碗要不要洗，厨房台面要不要擦。
- 走之前要不要把垃圾带走。

所以 AGENTS.md 就像提前贴在厨房门口的一张说明。它不是告诉 AI「这顿饭做什么菜」，而是告诉 AI「进这个厨房之前，先知道这里的规矩」。

可以在口播里把它说得轻一点：

> 不写这个文件，就像你请了一个执行力超强的人来家里做饭，但没告诉他厨房规矩。结果他不是不努力，他是努力得很吓人。

## Prompt 和 AGENTS.md 的区别

- **Prompt**：这一次你具体吩咐它做什么，比如「帮我炒个青菜」。
- **AGENTS.md**：它进厨房动手前，默认要知道哪些规矩，比如米在哪、哪些菜别动、用哪个锅、走前要不要带垃圾。

可以用一句话讲：

> Prompt 是临时吩咐，AGENTS.md 是动手前默认要遵守的厨房规矩。

## 前后对比

### 没写 AGENTS.md 时

AI Agent 进到你的文件里以后主要靠猜。用厨房那个比喻来说，就是它已经进门了，但不知道哪里能动、哪里不能动。

它可能不知道：

- 先打开哪个柜子。
- 米在哪个柜子，盐在哪个盒子。
- 冰箱里哪些菜是今晚要吃的，哪些菜先不能动。
- 该用哪瓶酱油。
- 哪个锅是炒菜的，哪个锅是煮汤的。
- 你到底喜欢清淡一点，还是喜欢重口一点。
- 吃完以后碗要不要洗，厨房台面要不要擦。
- 走之前要不要把垃圾带走。

这时候的问题不是 AI 不努力，而是它努力的方向可能不对。

### 写了 AGENTS.md 之后

AI Agent 进来先读规则。用厨房那个比喻来说，就是先看一眼「厨房说明」再动手。

它会知道：

- 哪个柜子放主食，哪个柜子放调料。
- 冰箱里今晚要吃哪些菜，哪些菜先不要动。
- 做这顿饭该用哪瓶酱油。
- 哪个锅拿来炒菜，哪个锅拿来煮汤。
- 做饭过程中遇到不确定的东西，先问你。
- 吃完以后要洗碗、擦台面。
- 走之前要顺手把垃圾带走。

它就从「一个聪明但陌生的人」，变成「知道你家厨房规矩的帮手」。

## 生活场景讲解

建议用「请人来家里做饭」作为主场景，不要用工作项目、自媒体项目、脚本资料这些例子。

可以这样讲：

> 你请一个人来家里做饭，他再厉害，也得先知道：哪个柜子里是米，哪个盒子里是盐；冰箱里哪些菜今晚要吃，哪些菜先别动；该用哪瓶酱油；哪个锅是炒菜的，哪个锅是煮汤的；吃完以后碗要不要洗，台面要不要擦；走之前要不要把垃圾带走。
> AGENTS.md 就是把这些话提前写下来。

也可以补一个更搞笑的表演点：

- 你扮演「没看说明书的 AI Agent」。
- 一本正经地站在厨房里很积极，但每一步都先问错地方：米在哪、盐在哪、这瓶酱油能不能用、这个锅是不是炒菜的。
- 然后切回正常状态：不是它笨，是你没给规矩。

## 怎么使用

这条先不展开复杂写法，只讲三个层级：

1. **背景说明**：写清楚这个地方是干什么的。比如「我请了一个人来帮我做饭」，这个文档是对方动手前需要先看的。
2. **操作环境和资源**：写清楚东西都放在哪里、哪些能用、哪些先不要动。比如米在哪、盐在哪、冰箱里有什么、炒菜用什么酱油、锅在哪里，做饭之前先看今天的菜单。
3. **收尾工作**：写清楚不确定时先问谁，以及任务结束后要做哪些清理和交接。比如不知道能不能用的东西先问你，最后洗碗、擦台面、把垃圾带走。

可以简单展示一个 10 行以内的示例，不要展示太复杂的真实项目规则。

示例方向：

```md
# 家里厨房规则

## 背景

- 我请你来家里帮我做饭。
- 这个文档是你进厨房动手前需要先看的。

## 操作环境和资源

- 做饭前，先看今天的菜单。
- 米在左边柜子，盐在灶台旁边的小盒子。
- 冰箱第一层是今晚要吃的菜，第二层先不要动。
- 炒菜用左边那瓶酱油。
- 黑色铁锅炒菜，不锈钢锅煮汤。

## 收尾工作

- 如果看到没写清楚的东西，先停一下问我，不要自己猜。
- 做完饭之后，把碗洗了，厨房台面擦干净。
- 离开前检查一下厨房，把垃圾带走。
```

## 拍摄方式

- 对标「秋芝2046」：认真讲一个有点荒诞的生活比喻，不做严肃教程腔；参考样本已归档到 `素材库/参考视频/秋芝2046/`。
- 开头前 3-15 秒必须遵守：强判断 / 强结果先行，生活画面承接，最后再点出 AGENTS.md，不要解释型开场。
- 真人口播为主，屏幕录制为辅。
- 真人部分负责解释、类比、吐槽和节奏。
- 录屏部分只作为证据：让观众看到 AGENTS.md 长什么样、里面大概写哪几类规则。
- 不要长时间讲界面按钮，不做安装、登录、配置、官网下载教学。
- 节奏可以快一点，字幕带一点吐槽感。
- 表现可以更活泼，有一点表演感，但不要变成纯搞笑段子，底层还是讲清楚概念。

## 需要注意的点

- 不要把 AGENTS.md 讲得太技术，核心比喻用「厨房使用说明书」。
- 不要把重点放在文件名本身，重点是「AI Agent 干活前要先知道项目规矩」。
- 不要用自媒体项目、脚本文件、素材库这些例子来解释，必须优先用生活场景。
- 不要讲成「每个人都必须写 AGENTS.md」，可以说适合长期项目、重复任务、需要稳定协作的场景。
- 注意外网工具教程风控：不讲官网、下载、安装、登录、账号配置等内容。
