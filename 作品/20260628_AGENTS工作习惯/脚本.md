## 脚本文案（拍摄版，剪映分批录制）

你好啊朋友们，今天这期可能会让你的 AI Agent 少犯一半的傻。很多人用 Agent 的方式，就像请人来家里做饭，但不告诉他米在哪、盐在哪、哪口锅能用。这个人越积极，你越害怕。因为他不是不会干活，他是太会干活了。所以在让 AI Agent 真正动手之前，你得先给它一份规矩。这个规矩，就叫 AGENTS.md。

你可以把 AGENTS.md 理解成你家厨房的使用说明书。你请一个人来家里帮你做饭，他再厉害，也得先知道几个事。哪个柜子里是米，哪个盒子里是盐。冰箱里哪些菜是今晚要吃的，哪些菜先不要动。该用哪瓶酱油。哪个锅是炒菜的，哪个锅是煮汤的。吃完以后碗要不要洗，厨房台面要不要擦。走之前要不要把垃圾带走。AGENTS.md 干的就是这件事。它不是告诉 AI 这一次要做什么，而是告诉它，进这个项目之前，默认要遵守什么规矩。

诶，那这里有朋友就有问题了，那我每次直接用提示词告诉AI不就好了吗，听起来就是多说几句话的事嘛，其实 提示词 和 AGENTS.md 还是有一些区别的。提示词 是你这一次具体吩咐它做什么，比如帮我炒个青菜，帮我刷个碗。

那AGENTS.md 是它动手前默认要知道的厨房规矩，比如米在哪、哪些菜别动、用哪个锅、走之前要不要带垃圾。没有这个文件，AI Agent 进到你的项目里就只能靠猜。它可能不知道先看哪个文件，哪些能改，哪些不能动，做完以后要不要检查。它不是不努力，是努力方向可能不对。

那AGENTS.md这个文件到底怎么写？其实不用很复杂，一共就三步。

第一步，背景说明，告诉AI这个地方是干什么的。就像你告诉做饭的人，我请你来家里帮我做饭，这个文档是你进厨房前先看的。

第二步，操作环境和资源，告诉AI东西放在哪里，哪些能用，哪些先不要动，然后它自己就去查了。

第三步，收尾工作，告诉它不确定的时候先问谁，做完以后怎么检查和交接。

其实AGENTS.md更像贴在门口的一张规矩，你想让 AI Agent 不产生幻觉，更不想频繁的去纠正，就得先把这些信息写下来。
