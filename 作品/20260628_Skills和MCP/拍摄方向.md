# C22｜Skills / MCP 是什么

## 拍摄定位

- **系列位置**：Codex 官方最佳实践第 5 条，作为这一组里最进阶的一条。
- **核心问题**：当你不只是偶尔问 AI，而是反复让它做同一类任务、连接真实工具和资料时，就需要 Skills 和 MCP。
- **一句话方向**：Skill 让 AI 记住固定流程，MCP/连接器让 AI 能接触真实工具和资料。

## 切入点和方向

- 从一个反复做的工作/学习流程切入，不从概念解释开始。
- 适合的案例：
  - 工作方向：每周整理会议纪要，提炼待办、风险和下周重点。
  - 学习方向：每次读完文章，都自动输出摘要、关键概念、复习题和行动建议。
  - 资料方向：固定收集一批资料，按同一格式整理成报告。
  - 工具连接方向：连接文档、表格、日历或任务系统，形成一个完整工作流。
- 讲法上可以分两层：
  - **Skill**：把重复工作写成 AI 能照着做的流程。
  - **MCP/连接器**：让 AI 能读取或操作外部工具，不只靠你复制粘贴。
- 这条可以承接 C18「插件实际应用」、C07「演示一次工作生成 Skill」、C10「Agent 和飞书联动」，但本条先讲底层理解，不急着把所有案例都塞进去。

## 需要注意的点

- 不要变成名词解释，必须绑定一个具体可见的任务结果。
- 不要做插件清单，不讲「装了哪些插件」，而讲「多了哪些真实能力」。
- 不要演示账号登录、安装、官网入口和环境配置。
- 对外表达时可以弱化 MCP 这个词，先说「连接工具」或「连接资料来源」，再补充专业名称。
- 注意权限和隐私边界：连接真实工具时，只展示脱敏内容或演示数据。
