# 自媒体内容创作要求

## 一、创作者定位

### 1.1 身份定位

- **8 年教育行业老兵**（低幼到初中全段经验）
- **AI 学习实践者**（用 AI 帮普通人提升学习能力）
- **擅长打造个人 IP 的自媒体创作者**

### 1.2 内容风格要求

- ✅ **自然生活化**：像朋友聊天一样，不刻意
- ✅ **无说教感**：避免"你应该"、"你必须"这类说法
- ✅ **无爹味**：不说教、不居高临下
- ✅ **真实表达**：用"我见过"、"我之前带过"这类亲身经历的说法

### 1.3 创作流程

> ⚠️ **核心原则：AI 不能自己编内容，必须基于创作者的真实经验写脚本。**
>
> 创作者的 8 年教育行业经验（低幼到初中）、运营实战经历、面试官视角、以及 AI 深度使用经验是内容的核心竞争力，AI 编不出真实的细节和案例。AI 的角色是帮助组织结构、润色语言、补充框架，而不是代替创作者输出观点。

**脚本创作的正确流程（每次写脚本必须完整走完，不可跳过任何一步）：**

1. **素材沟通（必须）**：AI 先和创作者沟通，了解这个选题创作者想讲什么、有哪些真实经历和案例。如果创作者没有主动说，**AI 必须主动提问**，例如：
  - "这个选题你自己最想讲的核心观点是什么？"
  - "你有没有相关的真实经历或案例？"
  - "你见过的最典型的正面/反面例子是什么？"
2. **搜集调研（必须）**：拿到创作者的想法后，AI 去搜集国内自媒体平台（小红书、抖音）上同类选题的内容，重点分析：
  - 同类爆款视频的选题角度、标题、钩子、结构
  - 哪些点是观众最关心的（看评论区高赞留言）
  - 哪些点已经被讲烂了（需要差异化）、哪些点别人没讲到（机会点）
3. **可行性讨论（必须）**：AI 把调研结果整理后和创作者讨论，核心回答以下问题：
  - 创作者的观点/角度和市面上已有内容相比，有没有差异化？
  - 有哪些点可以借鉴爆款经验优化？
  - 最终确认脚本的核心观点、结构和案例
4. **脚本初稿**：基于沟通确认的方向 + 创作者真实素材 + 爆款技巧，组织成脚本
5. **迭代打磨**：创作者审核，AI 根据反馈修改（脚本到 80 分就可以拍，不过度打磨）

### 1.4 拍摄方式

- **全程真人口播拍摄**
- **手机自拍视角**（拉近距离，像跟朋友视频聊天）

## 二、爆款短视频特点总结

### 2.1 开头3秒钩子公式（必用）

> ⚠️ **核心原则：前 3 秒信息密度要高，每一秒都要传递有效信息，不浪费任何一帧。**
>
> 数据显示：前 3 秒留存率每提升 1%，整体完播率增加 8%-12%。含具体数字的开头比泛泛描述的点击率高 28.6%。"哎你有没有发现"这类泛泛反问已严重同质化，观众免疫，应避免作为默认开头。

#### 主力开头（优先使用）


| 钩子类型      | 操作技巧              | 示例                              | 为什么有效                    |
| --------- | ----------------- | ------------------------------- | ------------------------ |
| **身份直击**  | 第一句亮身份 + 数字量化经验   | "我在教育行业做了 8 年""我做了8年运营，筛选过千份简历" | 1 秒建立信任，观众判断"这人有资格讲"     |
| **痛点场景化** | 直接描述目标人群正在经历的具体困境 | "春招到现在，简历投了几十份，一个面试都没有"         | 触发"这说的就是我"的代入感，完播率提升 42% |
| **反常识数据** | 用违反直觉的具体数字制造认知冲击  | "我在头部互联网公司，80%的同事不会真正用AI"       | 打破预设认知，好奇心驱动继续看          |
| **身份反差**  | 亮出与话题形成反差的身份，制造好奇 | "最近很多人用AI练面试，但我自己就是面试官"         | 身份反差制造"内行人视角"的稀缺感        |


#### 辅助开头（偶尔搭配使用）


| 钩子类型     | 操作技巧         | 示例                         | 注意事项                           |
| -------- | ------------ | -------------------------- | ------------------------------ |
| **利益前置** | 直接说看完能得到什么   | "掌握这3个技巧，你的简历通过率完全不一样"     | 利益要具体，不要空泛承诺                   |
| **损失规避** | 告知不看会失去什么    | "还在用去年的方法做账号，下个月可能失去80%流量" | 风险要真实可信，不要危言耸听                 |
| **具体反问** | 用有信息量的问题引发思考 | "你猜公司里有多少人真正会用AI？"         | 问题本身要有信息量，**禁用**"哎你有没有发现"等泛泛反问 |
| **反常理式** | 说反常识的话，引发好奇  | "买得起1000万别墅，吃不起30块快餐"      | 反差要真实，不要为了反差而编造                |


#### 3T 节奏模型（前 3 秒内部节奏）


| 时间      | 阶段                 | 任务             | 示例拆解                |
| ------- | ------------------ | -------------- | ------------------- |
| 0-0.8 秒 | **Trigger 触发**     | 视觉/语言冲击，让观众停下来 | "我做了8年运营"           |
| 0.8-2 秒 | **Tension 张力**     | 情绪升级，制造悬念或冲突   | "筛选过千份简历，面试官也当过不少回" |
| 2-3 秒   | **Transaction 交付** | 明确价值承诺，给出观看理由  | "今天从面试官视角跟你说几个简历的坑" |


### 2.2 5s完播率：流量生死线（数据实证）

> ⚠️ **血泪教训**：2026年3月，两条仅隔1天发布、同为 AI 方向的视频，因为5s完播率差了4个百分点（26.43% vs 22.11%），播放量差了5倍（1947 vs 391）。5s完播率是抖音算法决定是否给更大流量池的核心门槛。

#### 开头必须在5秒内完成"钩子投放"


| 原则         | 说明                            | 正面案例                                | 反面案例                                                    |
| ---------- | ----------------------------- | ----------------------------------- | ------------------------------------------------------- |
| **好奇缺口优先** | 先制造"想知道答案"的悬念，不要先给答案          | "用AI学东西，大部分人都少了最关键的一步"（5s完播率26.43%） | "2026年，会用AI的人涨薪8到10个点……"开头连抛3个数据，5秒内还没讲完钩子（5s完播率22.11%） |
| **一句话原则**  | 前5秒只说一句话，这句话的唯一任务是让人留下来       | "用AI学东西，大部分人都少了最关键的一步"              | 8秒才讲完的数据堆砌式开头                                           |
| **数据后置**   | 有冲击力的数据应该放在钩子之后，作为"证据"而非"开场白" | 先说"我发现了一个残酷真相"→再给数据                 | 上来就是"涨薪8到10个点""近八成公司……"                                 |


### 2.3 单一行动点原则（数据实证）

> ⚠️ **教训**：「AI学习最关键一步」给了一个具体方法（让AI考你），点赞39；「AI时代什么人最值钱」核心观点是"用AI解决手头的事"，太抽象，点赞仅9。


| 要求                    | 说明                                |
| --------------------- | --------------------------------- |
| **每条视频必须有且只有一个核心行动点** | 观众看完能立刻知道"我该做什么"                  |
| **行动点要具体、可操作**        | "让AI根据学的内容出题考你"✅ / "用AI解决你手头的事"❌  |
| **观众获得感 = 点赞动力**      | 具体方法给人"学到了"的感觉，抽象观点只给人"好有道理"但不会点赞 |


### 2.4 互动引导设计（数据实证）

> ⚠️ **教训**：「AI时代什么人最值钱」的结尾互动"你现在工作中有在用AI吗？"→ 评论0条。「AI学习最关键一步」的"你最近在用AI学什么？"→ 评论2条。零互动 = 告诉算法"这条不值得传播"。


| 原则             | 正面示例             | 反面示例                   |
| -------------- | ---------------- | ---------------------- |
| **问题要具体、容易回答** | "你最近在用AI学什么？"    | "你有在用AI吗？"（太宽泛，不知道怎么答） |
| **问题带争议性更好**   | "你觉得AI会取代你的岗位吗？" | "评论区聊聊"（没给方向）          |
| **降低回答门槛**     | "你中了几个？评论区打个数字"  | 需要组织长段文字才能回答的问题        |


### 2.5 内容结构要求

#### 结构 A：短平快型（约 2 分钟）

适用：单一观点、快速干货、信息密度高的内容


| 模块       | 要求                        | 时长        |
| -------- | ------------------------- | --------- |
| **开场钩子** | 5秒内用一句话制造好奇缺口，让人留下来       | 0:00-0:05 |
| **问题共鸣** | 描述目标人群正在经历的具体困境，触发"说的就是我" | 0:05-0:20 |
| **主体内容** | 每个点20-25秒，不拖沓             | 0:20-2:10 |
| **结尾升华** | 给希望 + 具体互动引导              | 2:10-2:20 |


#### 结构 B：深度价值型（4-5 分钟）⭐ 当前主力格式

适用：多个要点拆解、需要让观众「真正学会」而非只「知道」的内容


| 模块                 | 要求                                              | 时长        |
| ------------------ | ----------------------------------------------- | --------- |
| **痛点代入**           | 先用观众正在经历的具体场景引发共鸣（「你有没有这种感觉」），再亮身份背书            | 0:00-0:15 |
| **要点拆解（每个约 60 秒）** | 每个要点包含三层：**观点本身 → 为什么做不到（戳心理障碍）→ 具体怎么做（给操作步骤）** | 0:15-3:30 |
| **总结串联**           | 快速回顾所有要点                                        | 3:30-3:50 |
| **金句收束 + 签名**      | 最核心观点重申一句 → 「我是张半蛋，今天就讲这些，我们下期再见」收尾             | 3:50-4:20 |


**结构 B 的关键原则：**

- 开头**先代入再亮身份**，不要上来就自我介绍
- 每个要点的「为什么做不到」是关键，能降低观众"我早就知道了"的排斥感
- 「具体怎么做」要给到操作级别：打开什么 → 做什么 → 预期结果是什么
- 结尾**不做 CTA 号召**（详见第九章 9.1 禁用 3），用金句收束 + 签名收尾即可
- 互动引导放在**评论区置顶评论**里，不放在正片结尾（避免结尾讲完观点又让观众"去做什么"）
- 时长 4-5 分钟不是目标，**让观众有收获**才是目标，不为拉时长注水

### 2.6 爆款内容必备要素

✅ **真实小例子**：每个观点都要有具体场景、小故事
✅ **节奏紧凑**：不拖沓，每个点快速说完
✅ **给希望**：最后一定要告诉观众"可以改变"、"可以练习"
✅ **单一行动点**：每条视频给一个具体、可操作、马上能用的方法
✅ **互动引导**：结尾问具体且容易回答的问题
✅ **语言口语化**：像平时说话一样，不用书面语

## 三、脚本输出必备要素（必加）

每次输出脚本时，必须包含以下内容：

### 3.1 爆款封面文案

- **用途**：用于封面图，抓住用户眼球
- **要求**：
  - ✅ **短小精悍**：不一定要主副标题，一句话也可以
  - ✅ **视觉友好**：考虑小红书和自媒体主页的展示效果
  - ✅ **一眼看懂**：3秒内让用户知道讲什么
  - ✅ **抓眼球**：有冲击力、有好奇心、有痛点
  - ⚠️ **字数严格控制在 4-10 个字以内**：超过 10 个字缩略图里看不清
  - ⚠️ **最多两行字**：三行字在缩略图里必定拥挤，少即是多
- **技巧**：
  - 用数字（如"5个特征"、"3秒"）
  - 用问号/感叹号增加情绪
  - 戳中痛点或引发好奇
  - 关键词前置

### 3.2 短视频主标题（20字符以内）

- **用途**：视频发布时的标题
- **要求**：
  - ⚠️ **严格控制在20个字符以内**
  - 包含核心关键词
  - 有吸引力或引发好奇
  - 可用数字、问号、感叹号增加冲击力

### 3.3 爆款话题标签（4个）

- **用途**：增加视频曝光和流量
- **要求**：
  - ⚠️ **严格控制在4个标签**
  - 1个大话题（如#职场）
  - 2个中话题（如#工作能力、#职场新人）
  - 1个精准话题（如#职场技巧）
  - 标签要热门且与内容相关

### 3.4 即梦AI 封面图提示词（必加）

- **用途**：生成给即梦AI的提示词，让 AI 设计出带情绪表现力和文案排版的短视频封面图
- **生成流程**：
  1. 根据脚本内容，思考封面应该传达什么情绪（焦虑？惊讶？自信？无奈？）
  2. 设计创作者在封面中的具体动作和表情（手势、眼神、面部表情）
  3. 结合封面文案，规划文字的颜色、排版位置
  4. 组装成完整的即梦AI提示词
- **提示词模板**：

```
我要做一个短视频封面，写实人像，画面中心为一位[服装描述]的年轻亚洲男性，背景为[背景描述]。[表情描述]，[肢体动作描述]，表现出[与内容主题相关的情绪]。光影柔和自然，捕捉到细腻的情绪表达，画面具有高度的真实感，色彩纯净简约。文案是"[封面文案]"，文字排版：[文字颜色、大小、位置的具体要求]。
```

- **模板说明**：


| 模块          | 要求                     | 注意事项                          |
| ----------- | ---------------------- | ----------------------------- |
| **人物基础**    | 根据实际拍摄情况填写当天穿着和拍摄背景    | 写脚本时先确认拍摄环境，提示词要与实际画面一致       |
| **表情描述**    | 根据内容情绪设计，要具体到五官        | 如"眉头微蹙，眼神充满担忧""嘴角上扬，眼中带光"     |
| **肢体动作**    | 设计一个与内容相关的标志性动作        | 如"一只手扶着额头""双手摊开""竖起食指""指向镜头"  |
| **情绪关联**    | 动作和表情要与选题内容情绪一致        | 求职避坑→无奈叹息；AI学习→兴奋好奇；教育洞察→认真思考 |
| **光影风格**    | 固定：柔和自然、高真实感、色彩纯净简约    | 保持统一视觉风格                      |
| **文案排版**    | 指定每部分文字的颜色、粗细、位置       | 关键词用亮色（黄/红）突出，放在人物侧边留白处       |
| **文字行数**    | ⚠️ 最多两行，总字数 4-10 个字    | 三行字缩略图必糊。宁可少字大字号，也不要多字小字号     |
| **文字与人物关系** | 文字和人物各占一半画面，互不挤压       | 人偏右字在左 / 人偏左字在右，不要把字全堆在人物上方   |
| **颜色控制**    | 最多 3 种颜色：白 + 强调色 + 描边色 | 强调色用亮黄/橙/红，所有字加黑色描边确保清晰       |


- **常用情绪 × 动作参考**：


| 内容情绪  | 表情参考        | 动作参考                |
| ----- | ----------- | ------------------- |
| 警告/避坑 | 眉头微蹙，眼神严肃   | 手掌向前推出"停"的手势、双手交叉打× |
| 无奈/叹息 | 眉头紧锁，嘴角微微下拉 | 一只手扶额头、单手捂脸         |
| 惊讶/震撼 | 眼睛睁大，嘴巴微张   | 双手摊开、一只手捂嘴          |
| 自信/推荐 | 嘴角上扬，眼神坚定   | 竖起大拇指、双手抱胸          |
| 思考/洞察 | 眼神深邃，微微侧头   | 一只手托下巴、食指抵太阳穴       |
| 兴奋/好奇 | 眼中带光，表情生动   | 身体前倾、指向镜头           |
| 教学/讲解 | 表情认真，微微点头   | 竖起食指（重点强调）、手掌向上摊开   |


- **使用流程**：每次写脚本时，先和创作者确认本次拍摄的穿着和场景，再生成提示词。
- **示例**（选题：春招简历致命错误，当天穿白色卫衣，米色窗帘背景）：

> 我要做一个短视频封面，写实人像，画面中心为一位穿着白色卫衣的年轻亚洲男性，背景为米色窗帘。他眉头微蹙，眼神充满担忧，一只手扶着额头，表现出对糟糕简历的无奈和叹息。光影柔和自然，捕捉到细腻的情绪表达，画面具有高度的真实感，色彩纯净简约。文案是"简历踩这几个坑 直接没面试"，文字排版：将"简历踩这几个坑"用白色加粗，"直接没面试"用亮黄色或红色突出，放在人物身体侧边的留白处。

### 3.5 脚本文案的三个版本（必加，缺一不可）

每条脚本必须同时输出**三个版本**，放在同一个脚本文件里，分别用途不同：


| 版本              | 用途              | 格式要点                                                   |
| --------------- | --------------- | ------------------------------------------------------ |
| **剪辑标注版**       | 剪辑时查找跳剪点、花字、变焦点 | 台词 + 剪辑标注（`✂️` `🔤` `🔍` `🏷️` `🔊`），标注用 `>` 引用块单独成行   |
| **纯文案版**        | 对词、逐句核对、发布文案参考  | 把所有台词合并，去掉全部标注，**每句一段**（每句之间空行分开）                      |
| **拍摄版（剪映分批录制）** | 创作者用剪映分批次连续录制   | **分 4 段**（单段超过 60 秒可切 5 段），**段内一句接一句紧挨着**，**段与段之间空一行** |


#### 拍摄版的硬规则（重点）

> ⚠️ **用途说明**：创作者用剪映分批次录制，每批连续念一段不停顿。如果一段文字被换行切开，创作者会下意识停顿，影响录制节奏。
>
> **因此拍摄版必须做到**：段内句子紧挨着、不换行、不空行。

1. **分 4 段**：按内容逻辑自然切分，常见边界是「钩子 / 机制 / 方法 / 收束」
2. **段内紧挨**：句号直接接下句，不空行、不分行
3. **段间空行**：段落与段落之间必须空一行
4. **只保留正文**：**禁止**加批次标号、预估时长、情绪提示、拍摄要点、任何标注
5. **标点补完**：如果纯文案版里某句原本没句号（靠换行自带停顿），拼成连续段落时必须补句号，否则读的时候会断在错误的位置
6. **单段时长上限 60 秒**：超过 60 秒切分成两段（一般人连续念 60 秒以上容易忘词）

#### 格式示例

```markdown
## 脚本文案（拍摄版，剪映分批录制）

[第一段连续文本。下一句。再下一句。……]

[第二段连续文本。……]

[第三段连续文本。……]

[第四段连续文本。……]
```

## 五、视频展现形式与拍摄要求

### 5.1 内容形式选择指南

> 核心原则：不同内容类型用不同展现形式，发挥各自优势。你最强的武器是「真实的人 + 真实的经历」，形式选择不应削弱这个优势。

#### 形式 A：真人口播（主力形式）

适用场景：需要情绪感染力、故事感、建立个人信任的内容。


| 适用内容类型        | 举例                     |
| ------------- | ---------------------- |
| 沉浸式面试模拟       | 模拟面试系列（已验证的爆款引擎）       |
| 教育行业洞察 / 个人故事 | "在教育公司 8 年，我看到了什么"     |
| AI × 家庭教育     | "家长不懂英语，也能用 AI 辅导孩子口语" |
| 观点输出 / 认知类    | "AI 时代孩子最该培养的能力"       |


特点：

- 手机自拍视角，像跟朋友视频聊天
- 全程真人出镜，表情和情绪是核心吸引力
- 适合讲故事、分享经历、表达观点
- 个人品牌感最强，观众记住的是「你这个人」

#### 形式 B：大字报口播（辅助形式）

适用场景：信息密集、步骤清晰、方法论强的干货内容。


| 适用内容类型     | 举例                   |
| ---------- | -------------------- |
| 方法论 / 框架类  | "用 AI 学任何新技能的 3 步框架" |
| 干货清单类      | "工作能力差的 5 个特征"       |
| 工具对比 / 测评类 | "3 个最适合学习的 AI 工具"    |
| 步骤教程类      | "用 AI 做个性化学习计划的完整流程" |


特点：

- 画面以大字为主，真人缩小在角落或画面下方
- 文字 + 语音双通道传递信息，收藏率可能更高
- 制作效率高，适合快速产出
- 小红书生态天然适配（用户偏好信息密度高的画面）

操作要点：

- 字体要大、要粗，手机竖屏下一眼能看清
- 每屏文字不超过 3 行，关键词用亮色（黄/红）突出
- 真人出镜部分保持自然表情，不要完全变成「背景板」
- 语音节奏要配合文字切换，同步感很重要

#### 形式 C：录屏 + 口播（特定场景）

适用场景：需要展示 AI 工具实际操作的内容。


| 适用内容类型    | 举例                      |
| --------- | ----------------------- |
| AI 工具实操演示 | "我用 AI 自学 Python 的真实过程" |
| AI 学习效率对比 | "传统方法 vs AI 辅助，效率差多少"   |


特点：

- 录屏展示操作过程，口播讲解思路
- 真实感最强，「眼见为实」
- 必须严格遵守平台合规要求（见第七章）

#### 每次写脚本时的形式判断流程

1. 这个选题的核心是**讲故事/分享经历/表达观点**？→ 形式 A（真人口播）
2. 这个选题的核心是**传递方法论/列步骤/讲框架**？→ 形式 B（大字报口播）
3. 这个选题需要**展示 AI 工具实际操作过程**？→ 形式 C（录屏 + 口播）
4. 拿不准的时候 → 默认形式 A（真人口播永远不会错）

### 5.2 通用拍摄方式

- 📱 **手机自拍视角**：拉近跟观众的距离
- 🎬 **真人出镜**：不管哪种形式，都要有真人元素
- 🖼️ **简洁背景**：办公室或书房，不用太正式

### 5.3 表演要点

- 😊 **表情自然**：像跟朋友聊天，该吐槽吐槽，该共情共情
- ✋ **手势自然**：平时怎么说话就怎么用手势
- 🗣️ **语速适中**：不要太快也不要太慢，像平时说话
- 📝 **不要背稿**：熟悉内容，自然表达，忘词也没关系
- 💯 **真实最重要**：你真实，观众才会信你

## 七、平台合规与录屏避坑（必读）

> ⚠️ **血泪教训**：「AI练面试靠谱吗」视频因录屏展示了数据库英文代码，被平台判定为隐私/安全风险，两个平台同时未获推荐。以下规则每次涉及录屏的内容都必须检查。

### 7.1 录屏内容红线


| 绝对不能出现                          | 原因                              | 替代方案                          |
| ------------------------------- | ------------------------------- | ----------------------------- |
| **数据库代码 / SQL 语句 / 英文代码**       | 平台 OCR 扫描到代码和数据库内容，判定为隐私泄露或安全风险 | 口述说"我让 AI 自动连数据库取数"，不展示实际代码画面 |
| **公司名称、业务数据、用户数据**              | 侵权 + 隐私风险                       | 全部脱敏或打码，口述时用"某公司""某项目"替代      |
| **具体企业名 + 职级（如"阿里 P8"）**        | 可能被判定为虚假背书                      | 改为"大厂高级面试官"或只说职级不说公司          |
| **境外 AI 工具界面（Google Gemini 等）** | 国内平台对境外 AI 产品敏感                 | 用国产工具替代，或只口述不展示界面             |


### 7.2 录屏安全操作规范


| 规范                 | 说明                                      |
| ------------------ | --------------------------------------- |
| **录屏前先清理画面**       | 关闭无关标签页、隐藏敏感文件名、检查左侧目录有无公司相关文件夹名        |
| **不展示任何代码执行过程**    | AI 做数据分析、连数据库、写代码的过程只口述，展示最终结果（且结果也要脱敏） |
| **录屏时长不超过单次 15 秒** | 连续录屏太长会降低推荐权重（算法偏好人脸），每段录屏之间切回口播画面      |
| **录屏内文字字号调大**      | 缩小嵌入竖屏后观众要能看清关键信息                       |
| **发布前逐帧检查录屏画面**    | 确认没有遗漏的敏感信息                             |


### 7.3 AI / 职场类内容特别注意


| 场景                 | 风险            | 安全做法                   |
| ------------------ | ------------- | ---------------------- |
| 展示 AI 做数据分析        | 数据库、代码、业务数据泄露 | 只展示脱敏后的最终报告截图，过程口述     |
| 展示 AI 写代码          | 代码内容可能触发安全审核  | 展示最终产品效果，不展示代码过程       |
| 展示 Cursor / IDE 界面 | 文件目录可能暴露公司项目  | 用个人项目目录（如自媒体项目），不用工作项目 |
| 提到具体 AI 工具名        | 可能被判定为广告      | 口述提一嘴即可，不要反复强调品牌名      |
| 展示真实工作场景           | 公司信息泄露        | 用个人学习/自媒体场景替代工作场景      |

## 八、内容创作检查清单

每次创作前，检查以下几点：

**内容质量：**

- **素材来自创作者真实经验**：不是 AI 自己编的，每个案例都有真实出处
- **5秒内完成钩子投放**：用一句话制造好奇缺口，不要数据堆砌式开头
- **好奇缺口优先，数据后置**：先悬念再证据，不要上来就抛数字
- **单一行动点**：每条视频必须有一个具体、可操作、马上能用的方法
- **每个观点有真实小例子**：具体场景、小故事
- **语言口语化**：像聊天，不说教
- **无爹味**：不用"你应该"、"你必须"
- **节奏紧凑**：不拖沓，每个点20-25秒
- **结尾给希望**：告诉观众可以改变
- **互动引导要具体**：问容易回答的具体问题或有争议性的问题，不要太宽泛
- **真实表达**：用"我见过"、"我之前带过"

**发布物料（脚本中必须包含，缺一不可）：**

- **脚本三版本齐全**：剪辑标注版 + 纯文案版 + **拍摄版（剪映分批录制）**（详见 3.5）
- **爆款封面文案**：主标题+副标题
- **短视频主标题**：20字符以内
- **爆款话题标签**：4个标签
- **封面图完整设计方案**（每次写脚本必须同步输出，不可遗漏）：
  - 拍摄指南：表情（具体到五官）、动作（具体到手势和身体姿态）、视线方向、构图（人物位置和留白区域）
  - 文字排版示意：每行文字内容、颜色、大小层级、在画面中的位置
  - 即梦 AI 提示词：包含穿着、背景、表情、动作、构图、光影风格（文字部分不让 AI 生成）
  - 醒图/剪映加字步骤：具体操作步骤，确保创作者能直接执行

**平台合规（含录屏的视频必查）：**

- **无代码/数据库/SQL 画面**：AI 做数据分析的过程只口述，不录屏
- **无公司名称和业务数据**：全部脱敏或打码
- **无具体企业名+职级组合**：用"某大厂""高级面试官"替代
- **录屏单次不超过 15 秒**：每段录屏之间切回口播人脸
- **发布前逐帧检查录屏画面**：确认无遗漏敏感信息

## 九、创作者表达习惯（个人语感 · 铁律）

> ⚠️ **铁律说明**：以下是张半蛋本人的语感偏好，不是建议、不是参考，是**硬约束**。AI 写脚本时必须主动避开这些坑，发稿前必须按此清单自检。
>
> **提炼来源**：2026 年 4 月 18 日基于《AI 到底会替代谁》脚本 V1 的 11 处创作者手改提炼。本节持续扩充，每次创作者有新的语感偏好浮现，都补充进来。

### 9.1 禁用表达清单（硬规则）

#### 禁用 1：身体/机械比喻词（整体拒绝）


| 禁用词                     | 替代思路                     |
| ----------------------- | ------------------------ |
| **学习肌肉 / 肌肉记忆**         | 用「学习动力」「学习兴趣」「持续学习」这类直白词 |
| **认知带宽 / 心智带宽**         | 直接说「精力」「脑子转得过来」          |
| **飞轮 / 杠杆 / 引擎**（作为比喻）  | 直接讲机制本身，不套概念             |
| **骨架 / 血肉 / DNA**（作为比喻） | 换成「底层逻辑」「核心」这类中性词        |
| **护城河 / 天花板**（作为比喻）     | 直接说「别人追不上的优势」「能到的最高水平」   |


> **原因**：张半蛋语感偏平实克制，身体/机械比喻是典型"博主黑话"，会让内容带上"装逼感"。哪怕这个比喻流行也一样不用。
>
> **扩展原则**：凡是「把抽象概念套成具体物体」的比喻，都先过一遍"这是不是博主黑话"的自检。

#### 禁用 2：自指式铺垫与过渡


| 禁用句式            | 说明                      |
| --------------- | ----------------------- |
| 「这条视频我只讲一件事」    | 直接讲事，不预告自己要讲事           |
| 「真正要问的是一句话：XXX」 | 直接讲 XXX，去掉"要问的是一句话"的脚手架 |
| 「这个问题问错了」       | 既然问错了就不解释，给答案即可         |
| 「接下来我要告诉你…」     | 用「所以」一个字过渡              |
| 「听我把话说完」「你先别急」  | 不用任何留住观众的填充语            |
| 「记住这句话」「划重点」    | 金句就是金句，不标注金句            |


> **原因**：凡是"对内容的自我注解"都要删。观众只在乎内容本身，不在乎你要不要讲、准备怎么讲。

#### 禁用 3：结尾 CTA 号召（一律不做）

**硬规则**：**任何脚本的结尾都不做"行动号召"**，包括但不限于：

- ❌ 「今天你就试一件最小的事」
- ❌ 「明天上班第一个任务，先丢给 AI」
- ❌ 「现在就去做 XXX」
- ❌ 「看完这条立刻去 XXX」
- ❌ 「动手前先 XXX 一次」

**正确的结尾模式**（固定两步）：

1. **金句收束**：最核心的一句观点重申
2. **签名收尾**：「我是张半蛋，今天就讲这些，我们下期再见」

> **原因**：CTA 式结尾带"教你做"的姿态，和账号"平视沟通 / 观点给到就走"的气质冲突。
>
> **重要区分**（三者不一样，别搞混）：
>
> - ✅ **单一行动点**（脚本**中段**给一个具体可操作的方法）→ **保留**，这是内容的锚点
> - ✅ **互动引导**（评论区抛问题让观众选）→ **保留**，算法需要
> - ❌ **结尾 CTA**（结尾号召观众立刻去做某事）→ **禁用**

#### 禁用 4：俏皮形容词（观点位置）


| 场景   | 禁用              | 替代               |
| ---- | --------------- | ---------------- |
| 引出观点 | 讲了一个**挺狠的**道理   | 讲了一个**颠覆性的**道理   |
| 评价事物 | 这事儿**挺有意思的**    | 这事儿**值得琢磨**      |
| 强调程度 | **挺牛的 / 挺猛的**   | 用事实本身的分量说话，不加形容词 |
| 铺垫结论 | 我给你讲个**特别劲爆**的事 | 直接讲事             |


> **原因**：在"下观点"的位置上，要偏严肃克制。有分量的词比俏皮的词更可信。"挺狠的"这种口癖会稀释观点本身的分量。
>
> **注意**：这条只针对"下观点"的位置。日常对话、吐槽、代入场景时，口语俏皮是 OK 的。

### 9.2 写作习惯（减法原则）

#### 习惯 1：形容词能删就删


| 原文          | 改后        |
| ----------- | --------- |
| 文章**直接**说   | 文章说       |
| **特别朴素**的观察 | **特别**的观察 |
| 一个**侧**项目   | 一个项目      |
| 找人**搭**     | 找人        |
| **极其**重要    | 重要        |


**自检方法**：写完后把每个形容词/限定词逐个删一遍，如果删掉意思不变或只损失 5% 的精度，就删。

#### 习惯 2：短句干脆，过渡不绕

- **「所以。」 可以独立成句**（张半蛋改过的原句：「所以。AI 时代真正被淘汰的…」）
- 一个字能过渡就不用一句话
- 观点和观点之间，用句号分隔，不用"因此我想说""由此可见"

#### 习惯 3：用具体动作替代泛泛概念

用户手改里会把泛表达改成更具体的动作或对象。写学习类脚本时优先直接写观众正在做的事，不要停在抽象概念。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「不只是用来放松的」 | 「不只是用来刷手机放松的」 |
| 「晚上刚背完」 | 「晚上刚背完课文」 |
| 「新的信息马上盖上来 / 冲掉」 | 「新的信息马上覆盖前面学的东西」 |
| 「一个小内容」 | 「一段内容 / 一段很少的内容」 |

#### 习惯 4：少写模板味铺垫，多写个人口播转场

用户更接受轻一点、像临场分享的转场。可以用「今天想分享一个我个人的睡前习惯」「给 AI 的提示词我也整理出来了，可以截图保存」这类表达。少用「这条我想讲的不是...」「我的做法很简单」这类明显脚本模板味的句子；如果必须用，也要尽量压短。

#### 习惯 5：提示词表达要像真实操作，不像 AI 指令说明

写给 AI 的提示词时，优先用用户会直接复制的口语表达。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「你不要急着补充知识，先帮我判断」 | 「你来帮我判断三件事」 |
| 「请检查：哪些地方太空」 | 「第二，我哪里讲得太空」 |
| 「哪一句普通人最可能听不懂」 | 「哪一句普通人可能听不懂」 |
| 「不要立刻给你一篇标准答案」 | 「不要立刻给你一个标准答案」 |

#### 习惯 6：职场新人开头先讲现实关心，再转成长

写职场新人、实习生、校招生相关脚本时，不要默认用户一上来最在意成长。真实顺序通常是：钱给得到不到位、加班多不多、领导靠不靠谱、同事好不好相处；成长是过了两三个月之后才慢慢冒出来的问题。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「很多职场新人最难受的，不是加班，也不是工作杂」 | 「很多职场新人刚入职的时候，脑子里想的其实很现实。工资高不高，加班多不多，领导靠不靠谱，同事好不好相处？」 |
| 「每天都在干活，但说不清自己成长了什么」 | 「过了两三个月之后，另一个问题会慢慢冒出来：我好像每天都在忙，可我到底成长了什么？」 |
| 「这决定这段经历能不能变成能力」 | 「因为你可以忙三个月，也可以只是重复三个月」 |

#### 习惯 7：职场案例要像真实复盘，不像观点铺陈

职场经历类脚本优先先讲真实来源，再进入观点。案例表达要带具体时间、人物关系、事情细节，不要直接抽象成「我观察到」「我发现」。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「今天这条，是因为我和一个前实习生吃饭聊出来的」 | 「今天这条视频的灵感，是来自和之前带过的一个实习生吃饭聊出来的」 |
| 「他复盘的时候说了一个细节」 | 「她复盘的时候说了一个细节」 |
| 「我会跟他讲背景、目标和结果要求」 | 「我会跟他讲这件事的业务背景，目标是什么，有哪些关键动作，怎么做复盘」 |

#### 习惯 8：新人职场表达要具体到工作现场

写新人没人带、打杂、成长这类内容时，少用抽象判断，多用职场新人真实会遇到的动作和说法。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「先把手上这摊活接住」 | 「你需要先做很多很多的脏活和累活」 |
| 「按时交付」 | 「按时完成布置给你的工作」 |
| 「主动反馈进度」 | 「及时同步进度」 |
| 「做完之后能不能知道哪里做得好，哪里下次要改」 | 「做完之后会不会复盘」 |
| 「这段经历以后能不能变成能力」 | 「我活都干了，但我有了什么成长？可以丰富我的简历吗？」 |

#### 习惯 9：AI 辅助成长要落到真实用途

讲 AI 辅助职场成长时，不要只说「梳理业务链路」。要明确这件事对用户有什么用：理解工作、把杂活变成经验、写周报、转正答辩、写简历、面试复盘。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「AI 帮你把工作链路讲清楚」 | 「AI 帮你更好地理解工作」 |
| 「把杂活变成经验的开始」 | 「这就是你把杂活变成经验的开始」 |
| 「以后写周报、写转正总结、写简历、面试复盘都会用得上」 | 「以后写周报、转正答辩、写简历、面试复盘，都会用得上」 |
| 「用 AI 把你能接触到的工作链路讲清楚」 | 「用 AI 把你能接触到的工作链路想清楚」 |

#### 习惯 10：举"错误示范"时不写具体 AI 品牌名

脚本中举反面案例（如"大多数人怎么回答"）时，不要写"用豆包写文案""用 ChatGPT 改周报"这种具体品牌名。改成"用 AI 写方案""用 AI 做图"这类泛化表达。原因：①具体品牌名可能触发平台广告判定；②观众不一定用同款工具，泛化更有共鸣。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「用豆包写写文案，用 ChatGPT 改改周报」 | 「用 AI 写方案，做图，整理数据」 |

#### 习惯 11：身份表达默认简化

除非当条脚本需要强化"教育行业"背书，否则身份表达默认简化。用"我在互联网工作 8 年"而非"我在互联网教育公司做运营 8 年"。同理"这几年面试"比"这几年面试别人"更顺嘴。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「我在互联网教育公司做运营 8 年，这几年面试别人」 | 「我在互联网工作 8 年，这几年面试」 |

#### 习惯 12：直接说结果，不绕"出发→跑通"的过程描述

表达"做过 AI 项目"时，直接说结果更有力。不要用"从 XX 出发，用 AI 跑通了"这种过程描述。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「是从一个真实的工作问题出发，用 AI 跑通了」 | 「是用 AI 解决过一个真实的工作问题」 |
| 「你就接不住」 | 「你就答不上来」 |
| 「做过一个完整的事」 | 「做过一个完整的项目」 |
| 「不是试过功能」 | 「不是用过什么功能」 |

#### 习惯 13：连续列举用逗号，不用句号分隔

口播中连续列举几个短项时，用逗号连接更符合说话节奏。用句号会让每项听起来像独立的句子，不够连贯。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「我碰到了什么问题。我怎么用 AI 解决的。结果怎样」 | 「我碰到了什么问题，我怎么用 AI 解决的，结果怎样」 |

#### 习惯 14：不预设用户有特定行为习惯

不要替用户决定他会"记笔记""打开笔记本"。用更通用的表达，让不同习惯的观众都能代入。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「这些不是给别人看的，是给你自己存的。等面试的时候，打开笔记」 | 「等面试的时候，回想这部分内容」 |

#### 习惯 15：身份表达扩大到观众岗位，增加共鸣面

讲自己职业背景时，不只说自己的岗位，扩展到观众可能的岗位。"无论是做产品、做市场，还是做运营"比"做运营"覆盖面更广，观众更容易代入。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「做运营要一直跟着行业走」 | 「无论是做产品、做市场，还是做运营，都要一直跟着行业走」 |

#### 习惯 16：去掉"给你"减少说教感

"按类型给你分一下"里的"给你"带有居高临下感。直接说"按类型分一下"更平等。类似的"我来给你讲""给你推荐"都可以省掉"给你"。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「按类型给你分一下」 | 「按类型分一下」 |

#### 习惯 17：数字用范围比精确值更真实

口播中涉及时间、数量等数字时，用范围表达比精确值更贴近真实感受。"10-15 分钟"比"15 分钟"更可信。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「每天早上 15 分钟」 | 「每天早上 10-15 分钟」 |

#### 习惯 18：提到公众人物加"老师"

推荐别人的播客、书、课程时，提到公众人物名字要加"老师"表示尊重。同时不加个人主观评价词（如"不端着""很牛"），让观众自己判断。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「杨天真做的，聊职场选择，不端着」 | 「杨天真老师做的，聊职场选择和真实的人生认知」 |

#### 习惯 19：结尾签名前加一句祝福 + 结尾加"拜拜"

固定签名"我是张半蛋，今天就讲这些，我们下期再见"之前，可以加一句"希望今天的分享你能有些收获"让收口更温暖。签名之后加"拜拜"更亲切。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「我是张半蛋，今天就讲这些，我们下期再见」 | 「希望今天的分享你能有些收获，我是张半蛋，今天就讲这些，我们下期再见，拜拜」 |

#### 习惯 20：AI 工具类脚本要从个人使用问题切入，落到「干活」而不是概念

2026-06-16《别再只把 AI 当聊天机器人了》脚本手改新增：写豆包、Agent、办公任务这类内容时，用户更接受“我最近使用 AI 发现一个问题”这种个人使用观察开头，而不是“我最近发现一个明显变化”这类行业判断。工具价值要用普通工作语言说清楚：AI 帮你干活、自己去干了、把繁琐的事梳理出来；少用偏方法论的口号式表达。

| 原文倾向 | 更贴近用户的表达 |
| ------- | --------------- |
| 「我最近发现一个挺明显的变化」 | 「我最近使用 AI 发现一个问题」 |
| 「做一场活动复盘、整理一份调研报告」 | 「做一次复盘、整理一份报告」 |
| 「你把一个任务交代给它」 | 「你把一个任务交给它」 |
| 「体验一下 AI 帮我拆任务、跟任务、交结果」 | 「体验一下 AI 真正帮你干活是什么感觉」 |
| 「人就不想继续了」 | 「学习的意愿度就会降低」 |
| 「我不会只跟它说，帮我找几个选题」 | 直接给真实口播提示词，不额外解释太多 |
| 「我把四件事说清楚了：我是谁、我要做什么、我的受众是谁、最后我要拿到什么结果」 | 「把背景、目标、受众和最后要交付的东西说清楚，然后 AI 自己就去干活了」 |
| 「这一步很像我以前在教育公司做用户运营时带新人」 | 「其实使用 AI 很像我在公司带一个新人」 |
| 「你给它一句很大的问题」 | 「你给了一句含糊其辞的话」 |
| 「把一件乱糟糟的事先拆开」 | 「帮你解决一件复杂的事情」 |
| 「把一团乱的事情拆成几步」 | 「把一团乱麻、很繁琐的事情梳理出来」 |

**执行规则**：AI 工具类脚本少写“这条我想讲的不是某个工具有多强”“真正想说的是”这类自我解释；多用“最近 XX 可以试一下”“比如我要做 XX”直接进入场景。示例最好贴近本人真实工作：自媒体选题调研、复盘、报告、面试、学习计划。

### 9.3 使用方式

- **写脚本时**：在「脚本初稿」阶段就按本节清单自检，不要等到定稿才改
- **审稿时**：创作者审稿如果砍掉某个表达，AI 要判断属于「一次性偏好」还是「通用习惯」——属于通用习惯的就更新到本节，更新前**必须先和创作者确认**
- **迭代机制**：本节是**持续扩充**的，每次创作者有新的语感偏好浮现，都补充进来

## 十、素材库维护规则（铁律）

### 10.1 核心原则

**素材库 = 经过数据验证的经验沉淀，不是灵感记录本。**

- ✅ 灵感 → 放 `知识库/灵感速记.md`
- ✅ 进行中的选题 → 放 `知识库/选题池.md`
- ✅ 已写的脚本和发布后数据 → 放 `作品/{日期}_{标题}/`
- ✅ **经过 7 天数据验证、确认可复用的经验** → 才进 `素材库/`

混淆这四个层级，素材库就会被"自我感觉良好但没数据"的条目稀释，半年后失去参考价值。

### 10.2 素材库准入门槛（硬规则）

**时间门槛**：

- ❌ 发布当天 / 24h / 72h → **禁止**写入素材库
- ✅ 发布 ≥ 7 天 → 可评估是否达标
- ⚠️ 小红书长尾内容，数据还在增长 → 延到 14 天再评估

**数据门槛（小红书 / 抖音各自判断，三条中满足两条即达标）**：

1. 点赞 / 完播 / 每万观看涨粉 **≥ 账号同期均值 × 1.2**
2. 5 秒完播率 ≥ 40%（抖音硬门槛，过不了算法推荐直接算失败）
3. 评论 ≥ 5 条**真实共鸣**（不是"沙发""打卡""好棒"）

**金句特例**：如果某句话**被观众在评论区直接引用或截图**（即便整体数据不达标），可单独升级入「金句 & 案例库」，备注"观众引用"。

### 10.3 数据不达标的处理


| 情况             | 处理                                    |
| -------------- | ------------------------------------- |
| ✅ 双平台达标        | 全量提取 → 入素材库主区                         |
| ⚠️ 单平台达标       | 只提取达标平台对应的素材（封面/标题分开）                 |
| ❌ 双平台不达标，能说清原因 | 写进对应素材库的「反面教训」区                       |
| ❌ 双平台不达标，说不清原因 | 只在 `作品/{日期}/数据观察.md` 末尾标"数据不达标，未入素材库" |


### 10.4 标准工作流（发布到归档）


| 节点         | 动作            | 文档                  |
| ---------- | ------------- | ------------------- |
| 发布当天       | 填发布物料、观察节点    | `作品/{日期}/数据观察.md`   |
| 24h / 72h  | 记录数据快照        | `作品/{日期}/数据观察.md`   |
| **7 天节点**  | 按本章规则判定是否入素材库 | `数据观察.md` → `素材库/`* |
| 14 天节点（长尾） | 小红书二次确认       | 同上                  |


**禁止动作**：在发布当天或 7 天之前，把"感觉不错"的钩子 / 金句 / 封面公式写进素材库主区。

### 10.5 谁来执行

- `/cp-material-extract` Skill 必须先读 `数据观察.md`，数据不达标直接拒绝提取
- AI 若被要求"帮我现在提取一下"而时间不到 7 天 → 回复"距发布才 X 天，等数据出来再来"，**不帮用户破坏规则**
- 用户若坚持"就是要提" → 只允许写进「反面教训」区或候选观察笔记，**不得写入主区**
