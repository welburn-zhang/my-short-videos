# 运营数据表格命名规范

## 一、命名基本原则

- **数据时间优先**：文件名反映表格中的数据时间范围
- **平台明确**：清楚知道是哪个平台的数据
- **内容清晰**：知道表格里记录的是什么内容
- **版本管理**：支持持续更新，用版本号区分
- **便于查找**：按时间排序，快速定位

---

## 二、命名格式

### 2.1 日维度数据（单天）

```
YYYYMMDD_平台_日数据.扩展名
```

### 2.2 日维度数据（多天/时间范围）

```
开始日期_结束日期_平台_日数据.扩展名
YYYYMMDD_YYYYMMDD_平台_日数据.扩展名
```

### 2.3 持续更新版本（推荐）

```
开始日期_至今_平台_日数据_v版本号.扩展名
YYYYMMDD_至今_平台_日数据_v1.扩展名
```

---

## 三、详细命名说明

### 3.1 日期部分


| 类型       | 格式         | 说明      | 示例         |
| -------- | ---------- | ------- | ---------- |
| **单天日期** | `YYYYMMDD` | 年月日     | `20260302` |
| **开始日期** | `YYYYMMDD` | 数据起始日期  | `20260301` |
| **结束日期** | `YYYYMMDD` | 数据结束日期  | `20260331` |
| **至今**   | `至今`       | 持续更新到今天 | `至今`       |


### 3.2 平台部分


| 平台  | 命名  | 说明       |
| --- | --- | -------- |
| 小红书 | 小红书 | 全称，清晰易懂  |
| 视频号 | 视频号 | 全称，清晰易懂  |
| 抖音  | 抖音  | 全称，清晰易懂  |
| B站  | B站  | 全称，清晰易懂  |
| 全平台 | 全平台 | 多个平台汇总数据 |


### 3.3 数据类型部分


| 数据类型 | 命名   | 说明           |
| ---- | ---- | ------------ |
| 日数据  | 日数据  | 日维度的原始数据（推荐） |
| 每日数据 | 每日数据 | 每天的播放、点赞、评论等 |
| 周报   | 周报   | 每周汇总数据       |
| 月报   | 月报   | 每月汇总数据       |
| 粉丝数据 | 粉丝数据 | 粉丝增长、粉丝画像等   |
| 内容数据 | 内容数据 | 单个视频/笔记的详细数据 |
| 话题数据 | 话题数据 | 话题热度、相关话题等   |


### 3.4 版本号部分

- 格式：`v数字`（v1, v2, v3...）
- 用途：区分同一数据的不同更新版本
- 每次更新数据，版本号+1

## 五、工作流程建议

### 5.1 首次导入数据

1. 识别表格中的最早日期
2. 用这个日期作为开始日期
3. 命名为：`20260301_至今_小红书_日数据_v1.xlsx`

### 5.2 更新数据

1. 在原有表格中追加新数据
2. 识别最新的数据日期
3. 版本号+1，命名为：`20260301_至今_小红书_日数据_v2.xlsx`

### 5.3 月度归档

每月底，可以保存一个月度归档版本：

- `20260301_20260331_小红书_日数据.xlsx`
