# 学习测验流程

## 适用场景

- 用户在提问知识点、学习方法、概念解释、原理说明。
- 用户刚学完某个内容，希望被测验或复盘。

## 核心规则

- 当用户问知识性问题时，先讲解内容，**不要立刻出测验题**。
- 只有当用户明确表示已经学完、看完、可以考了，才开始出选择题测验。

触发词包括但不限于：
- "我学完了"
- "我看完了"
- "看好了"
- "学好了"
- "OK了"
- "可以考我了"
- "出题吧"

## 测验后流程

- 测验结束后，将知识点整理成笔记，存入 `学习笔记/` 对应分类目录。
- 生成测验报告，存入 `学习笔记/测验报告/`。
- 更新 `学习笔记/学习进度总览.md`。
