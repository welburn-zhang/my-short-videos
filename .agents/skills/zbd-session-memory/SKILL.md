# 对话备忘录机制

## 适用场景

- 项目相关讨论、决策、规划。
- 工作流、创作规范、长期策略的确认与沉淀。

## 新对话启动时

- 每次新对话开始，如果用户的问题涉及项目相关的讨论、决策、规划，先读取 `知识库/对话备忘录.md`，了解之前已达成的关键结论。

## 何时写入备忘录

- 在以下情况下，**主动提醒用户**是否需要记录到备忘录：
- 讨论出了新的策略、方向、决策
- 对某个问题达成了明确结论
- 确定了待办事项或下一步计划
- 修改了工作流程或创作规范

- 用户确认后才写入。

## 写入原则

- **只记结论，不记过程**：只写最终决策和关键共识，不写讨论的来回过程
- **一条结论一行**：简洁到能一眼扫完
- **标注日期**：每条记录标注日期，方便追溯
- **定期清理**：已落实到具体文件（如知识库文档、规则文件）的结论，从备忘录中移除，标记为「已归档」
- **分类清晰**：按主题分区，不要混成一锅粥

## 不该记录的内容

- 纯操作性的对话（如"帮我建个文件夹"）
- 已经直接写入其他文件的内容（避免重复）
- 学习过程中的知识点（这些走学习笔记流程）
