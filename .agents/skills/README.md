# Skills 目录说明

这个目录存放本项目的内容生产专属 Skills。

## 目录约定

- 所有可执行 skill 都直接放在 `.agents/skills/` 的第一层（每个 skill 一个文件夹 + SKILL.md）
- 需要分类时，优先用统一前缀命名，比如 `cp-*`（content production）
- 新增或删除 skill 后，顺手更新这个 README

## 当前结构

```text
.agents/skills/
├── README.md
├── cp-script-draft/     # 脚本初稿生成器
├── cp-topic-eval/       # 选题评估器
├── cp-material-extract/ # 素材提取器
├── data-weekly/         # 每周数据复盘器
└── short-video-illustration-storyboard/ # 短视频插画分镜
```

## 内容生产工具（cp-*）


| Skill                 | 触发词       | 功能                                |
| --------------------- | --------- | --------------------------------- |
| `cp-script-draft`     | 帮我写脚本/出初稿 | 给一个选题，生成精简脚本初稿（发布信息 + 双平台标题封面 + 拍摄版） |
| `cp-topic-eval`       | 帮我评估这个选题  | 评估一个选题的潜力，给出做/不做/调整后做的明确建议        |
| `cp-material-extract` | 帮我提取素材    | 发布后从脚本中提取可复用素材，格式化后贴入素材库          |


## 数据复盘工具（data-*）


| Skill         | 触发词               | 功能                                                                                    |
| ------------- | ----------------- | ------------------------------------------------------------------------------------- |
| `data-weekly` | 周报复盘/本周复盘/跑一下本周数据 | 读取 `运营数据/抖音` 和 `运营数据/小红书` 最新两版 xlsx，按 `复盘/模板_每周复盘.md` 格式产出周报，输出到 `复盘/YYYYMMDD_第N周.md` |


**使用前提**：每周日晚上，先在两个后台各导出一次新 xlsx，按命名规范放入对应目录（保留历史版本，不要覆盖），再触发 skill。

## 视觉分镜工具


| Skill                                | 触发词        | 功能                              |
| ------------------------------------ | ---------- | ------------------------------- |
| `short-video-illustration-storyboard` | 短视频插画分镜   | 根据脚本规划插画画面、插入时机、分镜图和生图/视频提示词 |
